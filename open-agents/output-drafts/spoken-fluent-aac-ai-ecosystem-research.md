## SPOKEN: AI & Prediction Technology Deep Dive

### The "Self-Learning Prediction" System

**What Wikipedia confirms**: "Spoken distinguishes itself from other AAC applications through the use of a **large language model** that provides predictive text capabilities, similar to a smartphone's autocomplete function." (Source: [Wikipedia - Spoken app](https://en.wikipedia.org/wiki/Spoken_(app)))

**How it works (based on all sources combined)**:
1. **Core mechanism**: Next-word prediction powered by AI/LLM -- the app suggests what you're likely to say next
2. **Self-learning**: "Every time you use the app, you're teaching it about your unique speech patterns and preferences" (Source: [Spoken Blog - Putting Predictions First](https://spokenaac.com/blog/putting_predictions_first/))
3. **Three ways to expand vocabulary**:
   - Use new words repeatedly in sentences so Spoken learns their context
   - Add terms in bulk via "Add Vocabulary" for basic prediction
   - Use "Personalized Predictions" feature -- answers targeted questions to ensure words are predicted in correct contexts
4. **Context cues**: Location (restaurant, school, hospital, shopping), time of day, audience, user preferences
5. **Handwriting/drawing recognition**: Data trained from Google's Quick, Draw! dataset

### Patent US11145313B2 -- "System and method for assisting communication through predictive speech"

- **Filed**: July 6, 2018 (US16/502,119)
- **Granted**: October 12, 2021
- **Assigned to**: SPOKEN INC. (assigned November 23, 2021)
- **Abstract**: "A database includes commonly used words, phrases, and images, each associated with at least one context cue. A processor is configured to determine the user's context and display a number of possible initial words, phrases, or images associated with the determined context."
- **Context cues include**: User preferences, time of day, location, audience, or combinations thereof
- **Key claim**: Text field updated with selected words/phrases/images; words spoken audibly
- **Technical approach**: Context-aware predictive generation -- NOT a specific neural architecture claim, but a system-level method for context-driven word/phrase suggestion
(Source: [Google Patents](https://patents.google.com/patent/US11145313B2/en))

### LLM Specifics -- What We Know and Don't Know

**Confirmed**:
- Uses a "large language model" (Wikipedia citation, multiple Spoken sources)
- The model is self-learning/adaptive to individual user patterns
- Context-aware with location, time, audience signals
- Predictions described as "cutting-edge artificial intelligence"
- Patent describes context-cue-based prediction at system level

**NOT confirmed / Unknown**:
- **Which specific LLM**: No public disclosure of whether it uses GPT, Llama, a custom model, or a fine-tuned open-source model
- **Architecture**: No confirmation of transformer vs. RNN vs. hybrid; "large language model" strongly suggests transformer-based
- **On-device vs. cloud**: NOT definitively confirmed. Privacy policy mentions anonymous data collection. The limitation that the app cannot be used fully offline suggests at least some cloud dependency for predictions.
- **Third-party LLM provider**: NOT confirmed. No evidence of GPT/Claude/Gemini branding. Could be proprietary or a fine-tuned open-source model.
- **Model size**: Unknown

**Assessment**: Spoken likely uses either a fine-tuned open-source LLM or a proprietary smaller language model. The "self-learning" component likely involves user-specific fine-tuning or adaptation layers on top of a base language model. The context-cue system (location, time, audience) feeds additional signals into prediction. The requirement for internet connectivity for full functionality suggests at least partial cloud processing.

### ElevenLabs Voice Model

- **Technology**: Neural network-based text-to-speech using "state-of-the-art neural networks to capture pitch, tone, accent, and rhythm" (Source: [ElevenLabs](https://elevenlabs.io/voice-cloning))
- **Cloning**: Requires only 10 seconds of voice audio
- **Voice design**: Text-prompt-based voice creation (no recording needed)
- **Model**: Eleven v3 is the latest model supporting Audio Tags for emotional control
- **Processing**: Cloud-based (ElevenLabs servers); requires internet connection
- **Emotional control**: Via Audio Tags ([sad], [angry], [happily], etc.) and stability slider

## OTHER AAC APPS & STARTUPS USING LLMs

### NWACS -- Alek (2025)
- **What it is**: New AI-powered AAC tool that listens to conversation and uses AI to suggest responses
- **Key innovation**: Context-aware -- tracks conversation history, knows user personality/preferences, suggests natural follow-up responses
- **Privacy approach**: On-device processing wherever possible; minimum data sent for suggestions; nothing used for advertising or stored long-term
- **Status**: Currently free (will have costs in future)
- **Significance**: Addresses the fundamental limitation that most AAC predictions don't consider conversation context
(Source: [NWACS Blog - June 2025](https://nwacs.info/blog/2025/6/why-most-aac-word-predictions-miss-the-mark-and-what-were-doing-about-it))

### SocializeChat (October 2025, Research)
- **What it is**: GPT-4-powered AAC tool grounded in personal memories to support social communication
- **Target users**: Elderly with speech impairments
- **Key innovation**: Uses personal memory database (events, routines, experiences) + persona profiles of conversation partners to generate contextually relevant suggestions
- **Technical**: Powered by ChatGPT-4; requires initial user database construction
- **Status**: Research paper (arXiv, October 2025)
(Source: [arXiv 2510.19017](https://arxiv.org/abs/2510.19017))

### KWickChat (Cambridge, 2022)
- **What it is**: Multi-turn dialogue system for AAC using bag-of-keywords
- **Technical**: GPT-2 language model with dialogue history and persona tags
- **Key result**: ~71% keystroke savings with word prediction
- **Significance**: Demonstrated that LLMs can generate full sentences from keyword input for AAC
(Source: [ACM IUI 2022](https://dl.acm.org/doi/10.1145/3490099.3511145))

### Google SpeakFaster Project
- **Published**: Nature Communications, November 2024
- **What it is**: LLM-powered UI for eye-gaze typing in highly-abbreviated form
- **Key results**: Text-entry rates **29-60% above baselines** for two ALS users; 57% more motor action savings than traditional predictive keyboards in offline simulation
- **Technical approach**:
  - Fine-tuned LLMs expand abbreviated text (word initials + supplementary letters) into full phrases
  - "KeywordAE" model: Expands abbreviations mixing initials with fully/incompletely spelled words
  - "FillMask" model: Provides alternative words beginning with given initial letter in context
  - Leverages conversational context for expansion
- **Significance**: THE landmark study demonstrating LLM-powered AAC can dramatically increase communication rates
(Source: [Nature Communications](https://www.nature.com/articles/s41467-024-53873-3); [Google Research Blog](https://research.google/blog/speakfaster-revolutionizing-communication-for-people-with-severe-motor-impairments/))

### PrAACT (2024, Expert Systems with Applications)
- **What it is**: Method using transformer-based language models (BERT) for communication card prediction
- **Key innovation**: Easy adaptability to user-specific vocabularies; domain adaptation through fine-tuning
- **Results**: Superior to pre-trained models in both zero-shot and few-shot scenarios
- **Significance**: Demonstrates that transformer models can be adapted for symbol-based AAC prediction
(Source: [ScienceDirect / Expert Systems with Applications](https://www.sciencedirect.com/science/article/abs/pii/S0957417423029196))

### Character-Based LLM Adaptation for AAC (January 2025)
- **What it is**: Research on adapting subword-token LLMs for character-level AAC prediction
- **Key finding**: Algorithm to produce character predictions from subword LLM provides more accurate predictions than classification layers, byte-level LLMs, or n-gram models
- **Domain adaptation**: Fine-tuned with curated dataset of AAC-useful sentences; curriculum approach effective
(Source: [arXiv 2501.10582](https://arxiv.org/abs/2501.10582))

### LLM + AAC Integration for Aphasia Rehabilitation (2024, PMC)
- SocializeChat concept using LLM for multi-turn conversations and tailored language rehabilitation
(Source: [PMC 10821375](https://pmc.ncbi.nlm.nih.gov/articles/PMC10821375/))

### The ASHA Debate: "Predictive Text: Who Controls the Conversation?" (March 2025)
- Published in ASHA Leader based on May 2024 "Future of AAC Research Summit" in Arlington, VA
- **Professionals' concern**: Predictive text may let technology control expression rather than the user
- **AAC users' response**: Professionals are "overstepping their bounds and becoming gatekeepers controlling access to technology"

- **Bottom line**: AAC users are largely excited about predictive text; professionals are more cautious
(Source: [ASHA Leader](https://leader.pubs.asha.org/do/10.1044/leader.FTR1.30032025.FAAC-predictive-text.36/full/))

## SPOKEN: Emotional Capabilities via ElevenLabs

### ElevenLabs Audio Tags in Spoken
- **How it works**: Audio Tags are words in square brackets interpreted by ElevenLabs v3 model to direct delivery
- **Emotional tags**: [sad], [angry], [happily], [sorrowful], [excitedly], etc.
- **Delivery tags**: Control pacing, emphasis, tone
- **In Spoken**: Can be set globally (all speech) or per-message
- **Limitation**: Currently requires typing brackets -- inaccessible to many AAC users
- **Planned improvement**: Spoken plans a tap-to-pick-tone interface
- **Significance**: "In verbal speech, how you say something matters just as much as what you say. Tone, pacing, emphasis, and delivery can communicate a lot. Audio tags offer a way to bring that color to AAC without being overly verbose."
(Source: [Spoken Blog - Audio Tags](https://spokenaac.com/blog/elevenlabs-audio-tags-in-aac/); [Spoken Help](https://spokenaac.com/help/how-to-use-elevenlabs-audio-tags-in-spoken/))

### ElevenLabs Voice Technology -- Emotional Capabilities
- **Neural network foundation**: Captures pitch, tone, accent, and rhythm
- **Emotional context from text**: Models interpret emotional cues from text input (e.g., "she said excitedly" or exclamation marks influence delivery)
- **Stability slider**: Lower stability = broader emotional range for the voice
- **Recommended model**: eleven_turbo_v2_5 for emotion and drama (dialogue, characters, storytelling)
- **Clone quality depends on training data**: Emotional nuance in the clone depends on variation in the source audio
(Source: [ElevenLabs Docs](https://elevenlabs.io/docs/overview/capabilities/text-to-speech); [ElevenLabs Blog](https://elevenlabs.io/blog/voice-cloning-deep-dive))

### Spoken: Adaptive UI Based on User State
- **Attention button**: Blinks device flashlight + plays customizable attention-grabbing sound (substitute for clearing throat)
- **Auto-attention**: "Get Attention When I Speak" setting plays alert automatically before speech
- **Context-based adaptation**: UI adapts predictions based on detected location (restaurant, hospital, school, etc.)
- **NO evidence found** of emotion detection from user behavior, facial expression, or biometric signals
- **NO adaptive UI based on emotional state** -- adaptation is currently environmental/contextual only

### Emotion-Adaptive AAC Startups/Products

**No dedicated emotion-adaptive AAC startup found.** This is a significant gap in the market.

# KEY FINDINGS FOR GRANT PROPOSAL

## 1. Gap Analysis: What Neither App Has
- **No SLP/clinician dashboard** or progress tracking
- **No caregiver/parent portal** for oversight or configuration
- **No data logging** for therapy measurement
- **No cloud sync/backup** confirmed for either
- **No school/district management** tools
- **No automatic emotion detection** from user behavior
- **No multi-modal emotion adaptation** (camera, biometric, behavioral)

## 2. LLM Opportunity
- Spoken confirms LLM usage but provides no specifics on architecture
- Fluent AAC has NO AI capabilities
- Google's SpeakFaster proved 29-60% rate improvement with LLMs (Nature Communications, 2024)
- The ASHA community has validated that AAC users want predictive text
- The field is moving rapidly toward LLM integration but no app has fully realized the potential
- Key research gap: Context-aware + emotion-aware + persona-aware prediction in a single system

## 3. Emotional Tone: Wide-Open Market
- Spoken's ElevenLabs Audio Tags are the only commercial emotion feature -- and they require manual text typing
- No AAC app detects user emotional state automatically
- Academic research confirms this is a desired future direction
- ElevenLabs v3 and competing voice synthesis (Marco-Voice, XTTS-v2, ECE-TTS) make the technology feasible
- The gap between "technology exists" and "AAC apps use it" is enormous

## 4. Ecosystem Weakness Across the Board
- Neither Spoken nor Fluent AAC has a meaningful ecosystem
- No companion apps, no clinician portals, no caregiver tools
- Proloquo2Go (AssistiveWare) is the ecosystem leader with Proloquo Coach companion app
- TD Snap (Tobii Dynavox) has the most complete institutional/school features
- There is a clear opportunity for a new entrant with a complete ecosystem from day one
