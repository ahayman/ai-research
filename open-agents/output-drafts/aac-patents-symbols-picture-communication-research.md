## EXECUTIVE SUMMARY

The AAC symbol and picture communication patent landscape is characterized by several critical IP barriers:

1. **Semantic Compaction / Minspeak Patents** -- The foundational Baker patents (US4,661,916; US5,097,425; US5,210,689) are now EXPIRED. However, Semantic Compaction Systems (now owned by PRC-Saltillo) has continued patenting improvements and owns newer active patents.

2. **Dynamic Keyboard Patents** -- US5,748,177 and US5,920,303 covering dynamic keyboard redefinition in AAC contexts are EXPIRED (2015 and 2017 respectively). These were the patents used to sue Speak for Yourself.

3. **Symbol Sets are NOT Patent-Protected** -- PCS, SymbolStix, Widgit, and Makaton symbols are protected primarily by **copyright** and **trademark**, NOT patents. This is a crucial distinction -- you cannot patent a drawing/symbol set, but you absolutely can copyright it.

4. **Tobii has a Massive Eye-Tracking Patent Portfolio** -- ~850 filings, 400+ granted patents covering eye tracking technology. This is relevant if your AAC device uses eye gaze.

5. **Open-Licensed Alternatives Exist** -- ARASAAC (13,000 symbols, CC BY-NC-SA), Mulberry (3,000 symbols, CC BY-SA), OpenMoji, and others provide free symbol alternatives, though with limitations.

6. **The biggest IP barriers for a new AAC entrant are NOT patents but rather copyright/trademark on established symbol sets** (PCS, SymbolStix) that clinicians and users already know.

## SEMANTIC COMPACTION / MINSPEAK PATENTS

### Overview
Minspeak is the patented system for Semantic Compaction, developed by linguist Bruce R. Baker. It uses multi-meaning icons (polysemic symbols) that code vocabulary in short sequences determined by rule-driven patterns. This was groundbreaking -- Baker was the first to develop the idea of sequencing picture symbols as an encoding system.

PRC-Saltillo acquired ALL intellectual property (patents, trademarks, and copyrights) related to Minspeak and Unity from Semantic Compaction Systems, Inc.

### The Landmark Lawsuit (2012)
This is the most important AAC patent litigation case in recent history:

- **Plaintiffs:** Prentke Romich Company & Semantic Compaction Systems
- **Defendant:** Speak for Yourself, LLC
- **Claims:** SfY infringed on US5,748,177 and US5,920,303 (dynamic keyboard patents)
- **Impact:** Despite no injunction being requested, Apple removed SfY from the App Store. Google also removed it from the Play Store. This silenced actual AAC users who relied on the app.
- **Settlement:** All parties settled. Semantic Compaction granted SfY a **non-exclusive license** to US5,748,177, US5,920,303, and certain Minspeak/Unity copyrights.
- **Terms:** Confidential.
- **Significance:** Demonstrated that AAC patent holders WILL enforce patents aggressively. The removal of the app from stores (without an injunction) showed how platform gatekeepers (Apple, Google) can amplify patent enforcement impact.

**Source:** https://www.techdirt.com/articles/20120326/08230818246/patents-threaten-to-silence-little-girl-literally.shtml

## KEY TAKEAWAYS & STRATEGIC IMPLICATIONS

### 1. Patent Risk Assessment

| Risk Area | Level | Notes |
|-----------|-------|-------|
| Semantic compaction / icon sequencing | **LOW** | Foundational Baker patents expired |
| Dynamic keyboard redefinition | **LOW** | US5,748,177 and US5,920,303 expired |
| Word/linguistic prediction in AAC | **MEDIUM** | PRC-Saltillo has active patents (US7,177,797 etc.) |
| Eye tracking for AAC | **HIGH** | Tobii has 400+ granted patents; licensing required |
| Network-connected AAC with AI suggestions | **MEDIUM** | Microsoft US10,262,555 covers this space |
| Specific device implementations | **MEDIUM** | PRC-Saltillo has 7+ active device patents |
| Motor planning / consistent word placement | **LOW** | Concept is not patented; specific implementations are copyrighted |

### 2. Copyright/Trademark Risk Assessment

| Risk Area | Level | Notes |
|-----------|-------|-------|
| Using PCS symbols | **VERY HIGH** | Must license from Tobii Dynavox |
| Using SymbolStix | **HIGH** | Must license from n2y/PRC-Saltillo |
| Using Widgit symbols | **HIGH** | Must license from Widgit Software |
| Using Makaton symbols | **HIGH** | Must get permission from The Makaton Charity |
| Calling your method "PECS" | **HIGH** | Trademark infringement |
| Creating your own original symbols | **NONE** | You own your own artwork |
| Using ARASAAC symbols | **LOW** | CC BY-NC-SA (watch non-commercial clause) |
| Using Mulberry symbols | **LOW** | CC BY-SA (commercially usable with attribution) |

### 3. Recommended Strategy for a New AAC Entrant

**Symbol Strategy:**
- **Develop proprietary symbols** -- this is the safest long-term approach and creates your own IP asset
- **Supplement with open-licensed symbols** (Mulberry for commercial use, ARASAAC if NC clause permits your use case)
- **License PCS from Tobii Dynavox** if clinical adoption is critical (PCS is the de facto standard)
- Support **Open Board Format (OBF)** for interoperability
- Reference the **W3C AAC Symbol Registry** for semantic mappings

**Patent Strategy:**
- The foundational AAC patents (semantic compaction, dynamic keyboards) are EXPIRED -- this opens the door
- **Avoid** linguistic prediction methods too similar to PRC-Saltillo's active patents
- If using **eye tracking**, must license from Tobii or use non-infringing technology
- If building **cloud-connected AAC with AI suggestions**, be aware of Microsoft's US10,262,555
- Consider filing your own patents on novel implementations (AI-powered symbol prediction, personalized learning algorithms, etc.)

**Litigation Risk:**
- PRC-Saltillo has a **demonstrated history** of aggressive patent enforcement (Speak for Yourself case)
- Apple and Google may remove apps from stores even without a court injunction, based on patent holder complaints
- Budget for IP legal review before launch

### 4. Opportunities in the Patent White Space

Areas where patents are expired or no dominant patents exist:
1. **AI-powered symbol generation** -- generating personalized symbols from photos/AI
2. **Symbol-to-speech with modern NLP/LLMs** -- using large language models for context-aware AAC
3. **Cross-platform AAC synchronization** -- real-time board sync across devices
4. **Adaptive symbol layouts** -- ML-driven personalization of symbol placement based on usage patterns
5. **Video Visual Scene Displays** -- emerging technology with limited patent coverage
6. **Speech-to-symbol translation** -- inverse AAC (understanding speech and displaying symbols)
