Deep Linking & State Restoration in SwiftUI

A comprehensive guide to handling external URLs, reconstructing navigation state, and persisting user journeys across app launches.

Deep linking lets users jump directly to specific content inside your app from external sources: web links, push notifications, Spotlight results, Siri suggestions, widgets, or other apps. State restoration ensures that when users return to your app after it has been terminated, they find themselves exactly where they left off.

In SwiftUI, these two capabilities are deeply intertwined. Both rely on NavigationPath serialization, both respond to external events, and both require careful coordination with your navigation architecture. This guide covers everything from basic URL handling to production-ready patterns that integrate with coordinators, tab views, and the full notification stack.

Prerequisites This guide assumes familiarity with SwiftUI's NavigationStack, NavigationPath, and the @Observable macro (iOS 17+). Code examples target iOS 17+ / Xcode 15+ unless noted otherwise.

Custom URL Schemes vs Universal Links

iOS supports two fundamentally different mechanisms for opening your app from a URL. Understanding their differences is critical before writing any deep-linking code.

Important: Both custom URL schemes and Universal Links are delivered to your SwiftUI code via the same .onOpenURL modifier. The difference is how the URL arrives at iOS, not how your app receives it.

How onOpenURL Works & Its Lifecycle

The .onOpenURL view modifier, introduced in iOS 14, is the primary SwiftUI API for receiving deep links. It fires whenever the system delivers a URL to your app, regardless of source.

When onOpenURL Is Called

  • App in foreground: Called immediately on the view that has the modifier
  • App in background: Called when the app returns to foreground after the URL opens it
  • App terminated: Called after the app launches and the scene is created

Multiple Handlers

You can attach .onOpenURL to multiple views in your hierarchy. All handlers fire for every URL — there is no "first responder" chain. This means you should design your URL handling so that only the appropriate handler acts on a given URL pattern.

Swift@main
struct MyApp: App {
    @State private var router = AppRouter()

    var body: some Scene {
        WindowGroup {
            ContentView()
                .environment(router)
                .onOpenURL { url in
                    // Central handler — parses and routes ALL deep links
                    router.handle(url: url)
                }
        }
    }
}
Best Practice: Use a single .onOpenURL at the root of your app (on WindowGroup's content view) and delegate to a centralized router. Avoid scattering handlers across child views unless you have a clear reason.

Custom URL Scheme Setup & Handling

Step 1: Register the URL Scheme in Info.plist

Add a CFBundleURLTypes entry to your app's Info.plist:

        </array>
    </dict>
</array>

Or in Xcode: Target → Info → URL Types → add a new entry with scheme myapp.

Step 2: Handle the URL with onOpenURL

Swiftimport SwiftUI

// MARK: - Route Definitions
enum AppRoute: Hashable, Codable {
    case productDetail(id: Int)
    case category(name: String)
    case userProfile(username: String)
    case settings
    case orderDetail(orderId: String)
}

// MARK: - URL Parser
struct DeepLinkParser {
    /// Parses URLs like:
    ///   myapp://products/42
    ///   myapp://categories/electronics
    ///   myapp://profile/johndoe
    ///   myapp://settings
    ///   myapp://orders/ABC-123
    static func parse(url: URL) -> AppRoute? {
        guard let components = URLComponents(url: url, resolvingAgainstBaseURL: false) else {
            return nil
        }

        let pathComponents = components.path
            .split(separator: "/")
            .map(String.init)

        // For custom schemes, host acts as the first path segment
        // myapp://products/42 → host = "products", path = "/42"
        guard let host = components.host else { return nil }

        switch host {
        case "products":
            guard let idStr = pathComponents.first,
                  let id = Int(idStr) else { return nil }
            return .productDetail(id: id)

        case "categories":
            guard let name = pathComponents.first else { return nil }
            return .category(name: name)

        case "profile":
            guard let username = pathComponents.first else { return nil }
            return .userProfile(username: username)

        case "settings":
            return .settings

        case "orders":
            guard let orderId = pathComponents.first else { return nil }
            return .orderDetail(orderId: orderId)

        default:
            return nil
        }
    }
}

// MARK: - App Entry Point
@main
struct MyApp: App {
    @State private var router = AppRouter()

    var body: some Scene {
        WindowGroup {
            RootView()
                .environment(router)
                .onOpenURL { url in
                    if let route = DeepLinkParser.parse(url: url) {
                        router.navigate(to: route)
                    }
                }
        }
    }
}

Universal Links with Associated Domains

Step 1: Create the Apple App Site Association (AASA) File

Host this JSON file at https://yourdomain.com/.well-known/apple-app-site-association (no file extension, served with Content-Type: application/json):

JSON{
    "applinks": {
        "apps": [],
        "details": [
            {
                "appID": "TEAMID.com.example.myapp",
                "paths": [
                    "/products/*",
                    "/categories/*",
                    "/profile/*",
                    "/orders/*"
                ]
            }
        ]
    }
}
Server Requirements:
  • Must be served over HTTPS with a valid certificate
  • No redirects allowed when fetching the file
  • Content-Type must be application/json
  • File size must be under 128KB

Step 2: Add Associated Domains Entitlement

In Xcode: Target → Signing & Capabilities → + Capability → Associated Domains. Add:

applinks:yourdomain.com

This also updates your .entitlements file:

Step 3: Handle Universal Links in SwiftUI

Swiftstruct UniversalLinkParser {
    /// Parses URLs like:
    ///   https://example.com/products/42
    ///   https://example.com/categories/electronics?sort=price
    static func parse(url: URL) -> AppRoute? {
        guard let components = URLComponents(url: url, resolvingAgainstBaseURL: false) else {
            return nil
        }

        // For universal links, use path components directly
        // https://example.com/products/42 → path = "/products/42"
        let pathSegments = components.path
            .split(separator: "/")
            .map(String.init)

        guard let firstSegment = pathSegments.first else { return nil }

        switch firstSegment {
        case "products":
            guard pathSegments.count > 1,
                  let id = Int(pathSegments[1]) else { return nil }
            return .productDetail(id: id)

        case "categories":
            guard pathSegments.count > 1 else { return nil }
            return .category(name: pathSegments[1])

        case "profile":
            guard pathSegments.count > 1 else { return nil }
            return .userProfile(username: pathSegments[1])

        case "orders":
            guard pathSegments.count > 1 else { return nil }
            return .orderDetail(orderId: pathSegments[1])

        default:
            return nil
        }
    }
}

// Unified handler that works with both scheme types
@main
struct MyApp: App {
    @State private var router = AppRouter()

    var body: some Scene {
        WindowGroup {
            RootView()
                .environment(router)
                .onOpenURL { url in
                    let route: AppRoute?
                    if url.scheme == "myapp" {
                        route = DeepLinkParser.parse(url: url)
                    } else {
                        route = UniversalLinkParser.parse(url: url)
                    }
                    if let route {
                        router.navigate(to: route)
                    }
                }
        }
    }
}

URL Parsing Strategies

Production apps need robust URL parsing that handles path components, query parameters, fragments, and edge cases. Here are several strategies.

    G --> H
    H --> I{"Route matched?"}

Strategy 1: Path-Based Matching

Swiftstruct PathBasedParser {
    static func parse(url: URL) -> AppRoute? {
        let segments = url.pathComponents.filter { $0 != "/" }

        switch segments {
        case ["products", let id] where Int(id) != nil:
            return .productDetail(id: Int(id)!)

        case ["categories", let name]:
            return .category(name: name)

        case ["profile", let user]:
            return .userProfile(username: user)

        case ["settings"]:
            return .settings

        default:
            return nil
        }
    }
}

Strategy 2: Query Parameter Extraction

Swiftstruct QueryParser {
    /// Handles: myapp://navigate?screen=product&id=42&ref=push
    static func parse(url: URL) -> AppRoute? {
        guard let components = URLComponents(url: url, resolvingAgainstBaseURL: false),
              let queryItems = components.queryItems else {
            return nil
        }

        let params = Dictionary(
            uniqueKeysWithValues: queryItems.compactMap { item in
                item.value.map { (item.name, $0) }
            }
        )

        guard let screen = params["screen"] else { return nil }

        switch screen {
        case "product":
            guard let idStr = params["id"], let id = Int(idStr) else { return nil }
            return .productDetail(id: id)

        case "category":
            guard let name = params["name"] else { return nil }
            return .category(name: name)

        case "profile":
            guard let username = params["user"] else { return nil }
            return .userProfile(username: username)

        case "order":
            guard let orderId = params["orderId"] else { return nil }
            return .orderDetail(orderId: orderId)

        default:
            return nil
        }
    }
}

Strategy 3: Regex-Based Route Matching (iOS 16+)

Swiftimport RegexBuilder

struct RegexRouter {
    struct RoutePattern {
        let regex: Regex<Substring>
        let handler: (String) -> AppRoute?
    }

    static let routes: [(pattern: String, handler: (String) -> AppRoute?)] = [
        ("^/products/(\\d+)$", { path in
            let regex = /^\/products\/(\d+)$/
            guard let match = path.wholeMatch(of: regex) else { return nil }
            return .productDetail(id: Int(match.1)!)
        }),
        ("^/categories/([\\w-]+)$", { path in
            let regex = /^\/categories\/([\w-]+)$/
            guard let match = path.wholeMatch(of: regex) else { return nil }
            return .category(name: String(match.1))
        }),
        ("^/profile/([\\w]+)$", { path in
            let regex = /^\/profile\/([\w]+)$/
            guard let match = path.wholeMatch(of: regex) else { return nil }
            return .userProfile(username: String(match.1))
        }),
    ]

    static func parse(url: URL) -> AppRoute? {
        let path = url.path()
        for route in routes {
            if let result = route.handler(path) {
                return result
            }
        }
        return nil
    }
}

NavigationPath Serialization

SwiftUI's NavigationPath has built-in support for encoding and decoding via its CodableRepresentation type. This is the foundation for both state restoration and deep linking.

How CodableRepresentation Works

When all types pushed onto a NavigationPath conform to Codable, the path exposes a codable property of type NavigationPath.CodableRepresentation?. This type-erased representation can be serialized to JSON and later decoded to reconstruct the exact same navigation stack.

Swiftimport SwiftUI

@Observable
class NavigationStore {
    var path = NavigationPath()

    private let savePath = URL.documentsDirectory
        .appending(path: "NavigationPath.json")

    func save() {
        guard let codable = path.codable else {
            print("NavigationPath contains non-Codable elements")
            return
        }

        do {
            let data = try JSONEncoder().encode(codable)
            try data.write(to: savePath, options: .atomic)
        } catch {
            print("Failed to save navigation path: \(error)")
        }
    }

    func restore() {
        do {
            let data = try Data(contentsOf: savePath)
            let decoded = try JSONDecoder().decode(
                NavigationPath.CodableRepresentation.self,
                from: data
            )
            path = NavigationPath(decoded)
        } catch {
            // No saved state or decoding failed — start fresh
            path = NavigationPath()
        }
    }
}

Making Your Routes Codable

Swift// All route types MUST conform to both Hashable and Codable
// for NavigationPath serialization to work.

enum AppRoute: Hashable, Codable {
    case productDetail(id: Int)
    case category(name: String)
    case userProfile(username: String)
    case settings
    case orderDetail(orderId: String)

    // Swift auto-synthesizes Codable for enums with associated values
    // as long as all associated value types are Codable.
}

// For more complex route data:
struct ProductRoute: Hashable, Codable {
    let id: Int
    let source: NavigationSource

    enum NavigationSource: String, Codable {
        case deepLink, search, browse, notification
    }
}

// Usage with NavigationStack:
struct ContentView: View {
    @State private var store = NavigationStore()

    var body: some View {
        NavigationStack(path: $store.path) {
            HomeView()
                .navigationDestination(for: AppRoute.self) { route in
                    switch route {
                    case .productDetail(let id):
                        ProductDetailView(productId: id)
                    case .category(let name):
                        CategoryView(name: name)
                    case .userProfile(let username):
                        ProfileView(username: username)
                    case .settings:
                        SettingsView()
                    case .orderDetail(let orderId):
                        OrderDetailView(orderId: orderId)
                    }
                }
        }
    }
}
Pitfall: If you push any non-Codable type onto a NavigationPath, the .codable property returns nil and you lose the ability to serialize the entire path. Always ensure every type you push conforms to Codable.

Reconstructing NavigationPath from URL Components

Swiftextension AppRouter {
    /// Builds a complete NavigationPath for a deep link, including
    /// any parent screens needed for proper back-navigation.
    func buildPath(for route: AppRoute) -> NavigationPath {
        var path = NavigationPath()

        switch route {
        case .productDetail(let id):
            // Product detail should have category in the back stack
            // Optionally: fetch product to know its category first
            path.append(route)

        case .orderDetail(let orderId):
            // Order detail should have order list in back stack
            path.append(AppRoute.settings) // settings contains order list
            path.append(route)

        default:
            path.append(route)
        }

        return path
    }

    func navigate(to route: AppRoute) {
        // Replace entire path for deep links (rather than appending)
        path = buildPath(for: route)
    }
}

SceneStorage for State Persistence

@SceneStorage is a property wrapper that persists simple values on a per-scene basis. It survives app termination and works perfectly for storing navigation state in multi-window environments (iPad, macOS).

Basic SceneStorage with NavigationPath

Swiftstruct ContentView: View {
    @SceneStorage("navigationPath") private var savedPathData: Data?
    @State private var path = NavigationPath()

    var body: some View {
        NavigationStack(path: $path) {
            HomeView()
                .navigationDestination(for: AppRoute.self) { route in
                    routeView(for: route)
                }
        }
        .onChange(of: path) {
            saveNavigationState()
        }
        .onAppear {
            restoreNavigationState()
        }
    }

    private func saveNavigationState() {
        guard let codable = path.codable else { return }
        savedPathData = try? JSONEncoder().encode(codable)
    }

    private func restoreNavigationState() {
        guard let data = savedPathData else { return }
        guard let decoded = try? JSONDecoder().decode(
            NavigationPath.CodableRepresentation.self,
            from: data
        ) else { return }
        path = NavigationPath(decoded)
    }

    @ViewBuilder
    private func routeView(for route: AppRoute) -> some View {
        switch route {
        case .productDetail(let id):
            ProductDetailView(productId: id)
        case .category(let name):
            CategoryView(name: name)
        case .userProfile(let username):
            ProfileView(username: username)
        case .settings:
            SettingsView()
        case .orderDetail(let orderId):
            OrderDetailView(orderId: orderId)
        }
    }
}

Restoring Navigation State After App Termination

Combining SceneStorage, NavigationPath, and your route model gives you full state restoration. Here is the pattern that handles both restoration and deep linking simultaneously.

Swiftimport SwiftUI

@Observable
class AppRouter {
    var path = NavigationPath()
    var selectedTab: AppTab = .home

    /// Called when a deep link arrives
    func handle(url: URL) {
        // Determine the route
        let route: AppRoute?
        if url.scheme == "myapp" {
            route = DeepLinkParser.parse(url: url)
        } else {
            route = UniversalLinkParser.parse(url: url)
        }

        guard let route else { return }

        // Switch to the appropriate tab first
        selectedTab = tab(for: route)

        // Slight delay to let the tab switch settle,
        // then set the navigation path
        Task { @MainActor in
            try? await Task.sleep(for: .milliseconds(100))
            path = NavigationPath()
            path.append(route)
        }
    }

    private func tab(for route: AppRoute) -> AppTab {
        switch route {
        case .productDetail, .category: return .home
        case .userProfile: return .profile
        case .settings, .orderDetail: return .settings
        }
    }
}

enum AppTab: String, CaseIterable, Codable {
    case home, search, profile, settings
}

// MARK: - Root View with State Restoration
struct RootView: View {
    @Environment(AppRouter.self) private var router
    @SceneStorage("savedTab") private var savedTab: String = AppTab.home.rawValue
    @SceneStorage("savedPath") private var savedPathData: Data?

    var body: some View {
        @Bindable var router = router

        TabView(selection: $router.selectedTab) {
            // Tab content here (see Tabbed Navigation section)
        }
        .onChange(of: router.selectedTab) { _, newTab in
            savedTab = newTab.rawValue
        }
        .onChange(of: router.path) {
            guard let codable = router.path.codable else { return }
            savedPathData = try? JSONEncoder().encode(codable)
        }
        .onAppear {
            restoreState()
        }
    }

    private func restoreState() {
        // Restore selected tab
        if let tab = AppTab(rawValue: savedTab) {
            router.selectedTab = tab
        }

        // Restore navigation path
        if let data = savedPathData,
           let decoded = try? JSONDecoder().decode(
               NavigationPath.CodableRepresentation.self,
               from: data
           ) {
            router.path = NavigationPath(decoded)
        }
    }
}
Tip: Deep links should replace the navigation path rather than appending to it. When a user taps a deep link, they expect to arrive at that specific screen, not have it stacked on top of wherever they were before.

Deep Linking with Tabbed Navigation

Tab-based apps need deep links that first switch to the correct tab, then navigate within that tab's NavigationStack. The cleanest approach gives each tab its own NavigationPath.

Swiftimport SwiftUI

@Observable
class TabRouter {
    var selectedTab: AppTab = .home
    var homePath = NavigationPath()
    var searchPath = NavigationPath()
    var profilePath = NavigationPath()
    var settingsPath = NavigationPath()

    func path(for tab: AppTab) -> NavigationPath {
        switch tab {
        case .home: return homePath
        case .search: return searchPath
        case .profile: return profilePath
        case .settings: return settingsPath
        }
    }

    func setPath(_ path: NavigationPath, for tab: AppTab) {
        switch tab {
        case .home: homePath = path
        case .search: searchPath = path
        case .profile: profilePath = path
        case .settings: settingsPath = path
        }
    }

    func navigate(to route: AppRoute) {
        let targetTab = tab(for: route)
        selectedTab = targetTab

        // Clear the tab's path and push the deep link route
        var newPath = NavigationPath()
        newPath.append(route)
        setPath(newPath, for: targetTab)
    }

    func popToRoot(tab: AppTab) {
        setPath(NavigationPath(), for: tab)
    }

    private func tab(for route: AppRoute) -> AppTab {
        switch route {
        case .productDetail, .category: return .home
        case .userProfile: return .profile
        case .settings, .orderDetail: return .settings
        }
    }
}

// MARK: - Tabbed Root View
struct TabbedRootView: View {
    @State private var router = TabRouter()

    var body: some View {
        @Bindable var router = router

        TabView(selection: $router.selectedTab) {
            Tab("Home", systemImage: "house", value: .home) {
                NavigationStack(path: $router.homePath) {
                    HomeView()
                        .navigationDestination(for: AppRoute.self) { route in
                            routeView(for: route)
                        }
                }
            }

            Tab("Search", systemImage: "magnifyingglass", value: .search) {
                NavigationStack(path: $router.searchPath) {
                    SearchView()
                        .navigationDestination(for: AppRoute.self) { route in
                            routeView(for: route)
                        }
                }
            }

            Tab("Profile", systemImage: "person", value: .profile) {
                NavigationStack(path: $router.profilePath) {
                    ProfileView(username: "me")
                        .navigationDestination(for: AppRoute.self) { route in
                            routeView(for: route)
                        }
                }
            }

            Tab("Settings", systemImage: "gear", value: .settings) {
                NavigationStack(path: $router.settingsPath) {
                    SettingsView()
                        .navigationDestination(for: AppRoute.self) { route in
                            routeView(for: route)
                        }
                }
            }
        }
        .environment(router)
        .onOpenURL { url in
            if let route = DeepLinkParser.parse(url: url) {
                router.navigate(to: route)
            }
        }
    }

    @ViewBuilder
    func routeView(for route: AppRoute) -> some View {
        switch route {
        case .productDetail(let id):
            ProductDetailView(productId: id)
        case .category(let name):
            CategoryView(name: name)
        case .userProfile(let username):
            ProfileView(username: username)
        case .settings:
            SettingsView()
        case .orderDetail(let orderId):
            OrderDetailView(orderId: orderId)
        }
    }
}

Push Notification → Deep Link Navigation

Push notifications commonly carry a deep link URL in their payload. When the user taps a notification, you extract the URL and route through the same pipeline.

Swiftimport SwiftUI
import UserNotifications

// MARK: - Notification Payload Example
// {
//   "aps": {
//     "alert": { "title": "Order Shipped!", "body": "Your order #ABC-123 is on its way." },
//     "sound": "default"
//   },
//   "deep_link": "myapp://orders/ABC-123"
// }

// MARK: - AppDelegate for Notification Handling
class AppDelegate: NSObject, UIApplicationDelegate, UNUserNotificationCenterDelegate {
    var router: AppRouter?

    func application(
        _ application: UIApplication,
        didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey: Any]? = nil
    ) -> Bool {
        UNUserNotificationCenter.current().delegate = self
        UNUserNotificationCenter.current().requestAuthorization(
            options: [.alert, .badge, .sound]
        ) { granted, error in
            if granted {
                DispatchQueue.main.async {
                    application.registerForRemoteNotifications()
                }
            }
        }
        return true
    }

    func application(
        _ application: UIApplication,
        didRegisterForRemoteNotificationsWithDeviceToken deviceToken: Data
    ) {
        let token = deviceToken.map { String(format: "%02.2hhx", $0) }.joined()
        print("APNs token: \(token)")
    }

    // Called when user taps notification while app is in foreground or background
    func userNotificationCenter(
        _ center: UNUserNotificationCenter,
        didReceive response: UNNotificationResponse,
        withCompletionHandler completionHandler: @escaping () -> Void
    ) {
        let userInfo = response.notification.request.content.userInfo

        if let deepLinkString = userInfo["deep_link"] as? String,
           let url = URL(string: deepLinkString) {
            Task { @MainActor in
                router?.handle(url: url)
            }
        }

        completionHandler()
    }

    // Show notification even when app is in foreground
    func userNotificationCenter(
        _ center: UNUserNotificationCenter,
        willPresent notification: UNNotification,
        withCompletionHandler completionHandler: @escaping (UNNotificationPresentationOptions) -> Void
    ) {
        completionHandler([.banner, .sound, .badge])
    }
}

// MARK: - App with Delegate
@main
struct MyApp: App {
    @UIApplicationDelegateAdaptor(AppDelegate.self) var appDelegate
    @State private var router = AppRouter()

    var body: some Scene {
        WindowGroup {
            RootView()
                .environment(router)

}
Why use AppDelegate? While SwiftUI's .onOpenURL handles URL-based deep links, push notification taps go through UNUserNotificationCenterDelegate. You need an AppDelegate adaptor to bridge this into your SwiftUI router.

struct RootView: View {

        // Handle Spotlight search results

            CSSearchableItemActionType
        ) { activity in
            guard let identifier = activity
                .userInfo?[CSSearchableItemActivityIdentifier] as? String
            else { return }

            // Parse identifier like "product-42"

            if parts.count == 2,

            router.navigate(to: .productDetail(id: id))

Deep Linking with Coordinators / Routers

The Coordinator pattern centralizes all navigation logic in a single object, making deep link handling predictable and testable.

Swiftimport SwiftUI

// MARK: - Comprehensive App Router (Coordinator)
@Observable
class AppRouter {
    var selectedTab: AppTab = .home
    var path = NavigationPath()
    var presentedSheet: AppSheet?
    var presentedAlert: AppAlert?

    // MARK: - Deep Link Handling
    func handle(url: URL) {
        let route: AppRoute?

        switch url.scheme {
        case "myapp":
            route = DeepLinkParser.parse(url: url)
        case "https", "http":
            route = UniversalLinkParser.parse(url: url)
        default:
            route = nil
        }

        guard let route else {
            print("Unhandled URL: \(url)")
            return
        }

        navigate(to: route)
    }

    // MARK: - Programmatic Navigation
    func navigate(to route: AppRoute) {
        // Dismiss any presented sheets first
        presentedSheet = nil

        // Switch to the correct tab
        selectedTab = tab(for: route)

        // Reset path and push the route
        // Use async to let tab switch settle
        Task { @MainActor in
            try? await Task.sleep(for: .milliseconds(50))
            path = NavigationPath()
            path.append(route)
        }
    }

    func push(_ route: AppRoute) {
        path.append(route)
    }

    func pop() {
        guard !path.isEmpty else { return }
        path.removeLast()
    }

    func popToRoot() {
        path = NavigationPath()
    }

    func present(sheet: AppSheet) {
        presentedSheet = sheet
    }

    // MARK: - Tab Mapping
    private func tab(for route: AppRoute) -> AppTab {
        switch route {
        case .productDetail, .category: return .home
        case .userProfile: return .profile
        case .settings, .orderDetail: return .settings
        }
    }
}

// MARK: - Sheet & Alert Types
enum AppSheet: Identifiable {
    case login
    case filter
    case share(item: String)

    var id: String {
        switch self {
        case .login: return "login"
        case .filter: return "filter"
        case .share(let item): return "share-\(item)"
        }
    }
}

enum AppAlert: Identifiable {
    case error(message: String)
    case confirmDelete(itemId: Int)

    var id: String {
        switch self {
        case .error(let msg): return "error-\(msg)"
        case .confirmDelete(let id): return "delete-\(id)"
        }
    }
}

// MARK: - Route & Tab Definitions
// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

enum AppRoute: Hashable, Codable {
    case productDetail(id: Int)
    case category(slug: String)
    case userProfile(username: String)
    case orderDetail(orderId: String)
    case settings
    case favoritesList

// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
// MARK: - Deep Link Parser
// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

enum DeepLink {
    /// Parses both custom scheme and universal link URLs.
    ///
    /// Supported patterns:
    ///   myapp://products/42
    ///   myapp://categories/shoes
    ///   myapp://profile/johndoe
    ///   myapp://orders/ORD-001
    ///   myapp://favorites
    ///   myapp://settings
    ///   https://example.com/products/42
    ///   https://example.com/categories/shoes
    static func parse(url: URL) -> AppRoute? {
        if url.scheme == "myapp" {
            return parseCustomScheme(url: url)
        } else {
            return parseUniversalLink(url: url)
        }
    }

    private static func parseCustomScheme(url: URL) -> AppRoute? {
        guard let host = url.host() else { return nil }
        let pathSegments = url.pathComponents.filter { $0 != "/" }

        switch host {
        case "products":
            guard let idStr = pathSegments.first,
                  let id = Int(idStr) else { return nil }
            return .productDetail(id: id)
        case "categories":
            guard let slug = pathSegments.first else { return nil }
            return .category(slug: slug)
        case "profile":
            guard let username = pathSegments.first else { return nil }
            return .userProfile(username: username)
        case "orders":
            guard let orderId = pathSegments.first else { return nil }
            return .orderDetail(orderId: orderId)
        case "favorites":
            return .favoritesList
        case "settings":
            return .settings
        default:
            return nil
        }
    }

    private static func parseUniversalLink(url: URL) -> AppRoute? {
        let segments = url.pathComponents.filter { $0 != "/" }
        guard let first = segments.first else { return nil }

        switch first {
        case "products":
            guard segments.count > 1, let id = Int(segments[1]) else { return nil }
            return .productDetail(id: id)
        case "categories":
            guard segments.count > 1 else { return nil }
            return .category(slug: segments[1])
        case "profile":
            guard segments.count > 1 else { return nil }
            return .userProfile(username: segments[1])
        case "orders":
            guard segments.count > 1 else { return nil }
            return .orderDetail(orderId: segments[1])
        default:
            return nil
        }
    }
}

// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
// MARK: - App Router (Coordinator)
// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

@Observable
class AppRouter {
    var selectedTab: AppTab = .home
    var path = NavigationPath()

    func handle(url: URL) {
        guard let route = DeepLink.parse(url: url) else {
            print("⚠️ Unrecognized deep link: \(url)")
            return
        }
        navigate(to: route)
    }

    func navigate(to route: AppRoute) {
        // Switch to the appropriate tab
        selectedTab = targetTab(for: route)

        // Reset and push
        Task { @MainActor in
            try? await Task.sleep(for: .milliseconds(50))
            path = NavigationPath()
            path.append(route)
        }
    }

    func push(_ route: AppRoute) {
        path.append(route)
    }

    func pop() {
        guard !path.isEmpty else { return }
        path.removeLast()
    }

    func popToRoot() {
        path = NavigationPath()
    }

    private func targetTab(for route: AppRoute) -> AppTab {
        switch route {
        case .productDetail, .category: return .home
        case .favoritesList: return .favorites
        case .userProfile: return .profile
        case .settings, .orderDetail: return .settings
        }
    }
}

// MARK: - Tab View with State Restoration
// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

struct AppTabView: View {
    @Environment(AppRouter.self) private var router
    @SceneStorage("selectedTab") private var savedTab: String = AppTab.home.rawValue
    @SceneStorage("navigationData") private var savedPathData: Data?

    var body: some View {
        @Bindable var router = router

        TabView(selection: $router.selectedTab) {
            ForEach(AppTab.allCases, id: \.self) { tab in
                Tab(tab.title, systemImage: tab.icon, value: tab) {
                    NavigationStack(path: $router.path) {
                        tabRootView(for: tab)
                            .navigationDestination(for: AppRoute.self) { route in
                                destinationView(for: route)
                            }
                    }
                }
            }
        }
        .onChange(of: router.selectedTab) { _, newTab in
            savedTab = newTab.rawValue
        }
        .onChange(of: router.path) {
            guard let codable = router.path.codable else { return }
            savedPathData = try? JSONEncoder().encode(codable)
        }
        .task {
            restoreState()
        }
    }

    private func restoreState() {
        if let tab = AppTab(rawValue: savedTab) {
            router.selectedTab = tab
        }
        if let data = savedPathData,
           let decoded = try? JSONDecoder().decode(
               NavigationPath.CodableRepresentation.self,
               from: data
           ) {
            router.path = NavigationPath(decoded)
        }
    }

    @ViewBuilder
    private func tabRootView(for tab: AppTab) -> some View {
        switch tab {
        case .home:
            HomeScreen()
        case .favorites:
            FavoritesScreen()
        case .profile:
            ProfileScreen()
        case .settings:
            SettingsScreen()
        }
    }

    @ViewBuilder
    private func destinationView(for route: AppRoute) -> some View {
        switch route {
        case .productDetail(let id):
            ProductDetailScreen(productId: id)
        case .category(let slug):
            CategoryScreen(slug: slug)
        case .userProfile(let username):
            UserProfileScreen(username: username)
        case .orderDetail(let orderId):
            OrderDetailScreen(orderId: orderId)
        case .settings:
            SettingsScreen()
        case .favoritesList:
            FavoritesScreen()
        }
    }
}

Tradeoff Analysis

URL Scheme Choice

Custom URL Schemes

  • Zero server setup required
  • Full Simulator support
  • Quick to implement and test
  • Works for app-to-app communication

Custom URL Schemes: Drawbacks

  • No uniqueness guarantee — another app can hijack your scheme
  • No web fallback if app is not installed
  • Cannot be used for App Clips
  • Apple discourages for user-facing links

Universal Links

  • Domain-verified ownership (secure)
  • Graceful web fallback
  • Required for App Clips
  • Better user experience (no "Open in..." dialog)

Universal Links: Drawbacks

  • Requires server with HTTPS and AASA file
  • Safari on Simulator does not trigger them
  • Debugging can be painful (caching, CDN issues)
  • Two-way association must be maintained

State Restoration Approach

ApproachProsCons
@SceneStorage Built-in, per-scene, survives termination Limited types, small data only, no encryption
UserDefaults Simple, shared across scenes Not per-scene, synchronous, no encryption
File-based (Documents) Unlimited size, full control Manual cleanup, not per-scene by default
Keychain Encrypted, survives reinstalls Complex API, not designed for navigation state

Single Path vs Per-Tab Paths

Single NavigationPath

  • Simpler to reason about
  • One place to serialize/restore
  • Works well for simple apps

Single NavigationPath: Drawbacks

  • Path resets when switching tabs
  • Cannot preserve per-tab navigation history
  • Deep links destroy current tab's state

Per-Tab NavigationPaths

  • Each tab retains its own back stack
  • Tab switching preserves navigation
  • More natural UX for tabbed apps

Per-Tab Paths: Drawbacks

  • More complex state management
  • Must serialize/restore multiple paths
  • Potential memory overhead for deep stacks

Router Architecture

Centralized Router

  • Single source of truth for navigation
  • Easy to test (just test the router)
  • Deep links route through one place
  • Consistent behavior across entry points

Centralized Router: Drawbacks

  • Can become a "god object" in large apps
  • Every view depends on the router
  • Harder to modularize into separate packages

Implementation Checklist

Use this checklist when adding deep linking and state restoration to an existing SwiftUI app.

Phase 1: Foundation

  • Define all routes as a Hashable, Codable enum
  • Create an @Observable router/coordinator class
  • Replace any NavigationLink(destination:) with NavigationLink(value:)
  • Use NavigationStack(path:) with your router's NavigationPath
  • Add .navigationDestination(for:) with a centralized switch

Phase 2: Custom URL Scheme

  • Register URL scheme in Info.plist (CFBundleURLTypes)
  • Write a URL parser that converts URLs to your route enum
  • Add .onOpenURL { } at the root of your app
  • Test with xcrun simctl openurl booted "myapp://..."
  • Write unit tests for every URL pattern

Phase 3: Universal Links

  • Create and host AASA file at /.well-known/apple-app-site-association
  • Verify AASA is served over HTTPS, no redirects, correct Content-Type
  • Add Associated Domains entitlement (applinks:yourdomain.com)
  • Extend your URL parser to handle https:// URLs
  • Test on a real device (Simulator Safari does not trigger Universal Links)
  • Validate with Apple's App Search Validation Tool

Phase 4: State Restoration

  • Verify all route types conform to Codable
  • Add @SceneStorage for NavigationPath data and selected tab
  • Save path on every .onChange(of: path)
  • Restore path in .onAppear or .task
  • Test: navigate deep, force-quit app, relaunch — verify state is restored
  • Test: open same app in two iPad windows — verify independent state

Phase 5: Notifications & Spotlight

  • Add UNUserNotificationCenterDelegate via @UIApplicationDelegateAdaptor
  • Parse deep_link field from notification payload in didReceive
  • Route notification URLs through your existing router
  • Index key content with CSSearchableItem
  • Handle Spotlight results with .onContinueUserActivity(CSSearchableItemActionType)
  • Advertise activities with .userActivity() modifier

Phase 6: Testing & Validation

  • Unit test every URL pattern (valid, invalid, edge cases)
  • Unit test router state transitions
  • Unit test NavigationPath serialization round-trip
  • UI test deep links with xcrun simctl openurl
  • Test deep link when app is not running (cold start)
  • Test deep link when app is in background
  • Test deep link when app is in foreground
  • Test Universal Links on a real device
  • Verify state restoration after force-quit
  • Test with multiple windows on iPad
