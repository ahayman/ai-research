Self-Supervised Foundation Models

Pre-trained on massive unlabeled audio corpora, these models learn rich speech representations that transfer powerfully to emotion recognition -- achieving the best accuracy with minimal labeled data.

Introduction

Self-supervised learning (SSL) has revolutionized speech processing. Rather than requiring millions of labeled emotion samples, these models first learn general speech representations from vast amounts of unlabeled audio, then transfer that knowledge to emotion recognition with remarkably little labeled data. The result: state-of-the-art accuracy that was unimaginable just a few years ago.

This category covers the current frontier of speech emotion recognition (SER). While traditional approaches (MFCCs + SVM) and supervised deep learning (CNNs, LSTMs) remain relevant for constrained deployments, self-supervised foundation models now define what is possible in terms of accuracy and generalization.

Key Insight

Foundation models like emotion2vec achieve state-of-the-art on IEMOCAP with only a linear classification layer on top of frozen pre-trained representations. No complex fine-tuning required -- the pre-trained features already encode emotion.

The Self-Supervised Learning Paradigm

Self-supervised learning solves the fundamental bottleneck in speech emotion recognition: the scarcity of labeled emotional speech data. While labeled datasets like IEMOCAP contain roughly 12 hours of data, SSL models pre-train on 960 to 60,000+ hours of unlabeled speech.

Why SSL Works for Emotion

  • Acoustic universals: Pitch contours, energy patterns, and spectral dynamics that distinguish emotions are present in all speech, not just labeled emotional data
  • Contextual learning: Transformer architectures learn long-range dependencies between acoustic features that correlate with emotional expression
  • Transfer power: Representations learned for speech understanding (phonemes, prosody, speaker identity) decompose naturally into features useful for emotion
  • Data efficiency: Once pre-trained, these models need only hundreds -- not thousands -- of labeled examples to achieve competitive emotion recognition

Three Pre-training Paradigms

Contrastive Learning

Example: Wav2Vec2

Learns by distinguishing true speech representations from distractors. Masks portions of the input and requires the model to identify the correct quantized target among negative samples.

Masked Prediction

Example: HuBERT

Masks input frames and predicts discrete cluster assignments (pseudo-labels from k-means). Iteratively refines both representations and targets across training rounds.

Online Distillation

Example: data2vec, emotion2vec

Teacher-student framework where both networks are initialized identically. The teacher (updated via EMA) produces targets from unmasked input; the student predicts these from masked input.

Wav2Vec2 uses a CNN to extract local features, quantizes them for contrastive targets, masks ~50% of frames, and trains a Transformer to identify the correct quantized target among distractors.

CNN Feature Encoder

Seven convolutional blocks with 512 channels each process the raw 16kHz waveform. The strides [5,2,2,2,2,2,2] and kernel widths [10,3,3,3,3,2,2] downsample the audio to approximately 49Hz -- one representation vector every ~20ms. Each block uses layer normalization and GELU activation.

Contrastive Pre-training

The contrastive lossA training objective that teaches the model to distinguish a correct target from incorrect distractors. The model maximizes cosine similarity with the true target while minimizing similarity with negatives. works as follows: for each masked time step, the model must identify the true quantized latent representation among K+1 candidates (1 true + K distractors sampled from other masked positions in the same utterance). The diversity lossA regularization term that maximizes the entropy of codebook usage, ensuring all quantized entries are used uniformly rather than collapsing to a few frequently-used codes. prevents codebook collapse by encouraging uniform usage of all quantization entries.

Emotion Recognition with Wav2Vec2

  • SpeechBrain pre-built model: wav2vec2-IEMOCAP achieves 78.7% accuracy (4-class) using attentive statistical pooling + Additive Margin Softmax
  • Frozen Feature Extractor: Freeze the CNN feature encoder while fine-tuning the Transformer and classification head -- preserves acoustic understanding
  • Combined architectures: Wav2vec2 + Neural Controlled Differential Equations (NCDE) achieves 73.37% WA, 74.18% UA (2025)
  • Best reported: 79.58% WA in speaker-dependent setting on IEMOCAP

HuBERT

HuBERT (Hidden-Unit BERT, Meta AI, 2021) shares the same architecture as Wav2Vec2 but takes a fundamentally different approach to generating training targets. Instead of learning targets on-the-fly through quantization (Gumbel-softmax), HuBERT uses offline k-means clustering to generate discrete pseudo-labels, then iteratively refines both the representations and the targets.

Key Difference: Offline Clustering

HuBERT's training alternates between two phases:

  1. Clustering phase: Run k-means on acoustic features (MFCCs in iteration 1, learned representations in subsequent iterations) to produce cluster assignments as pseudo-labels
  2. Prediction phase: Train the model with masked prediction -- predict the cluster assignment for each masked frame
Wav2Vec2 vs HuBERT: Target Generation

Wav2Vec2 learns its targets simultaneously during training via Gumbel-softmax quantization -- the targets and representations co-evolve.

HuBERT pre-computes targets offline via k-means clustering, then uses them as fixed labels during each training iteration. After training, new k-means clustering on the improved representations produces better targets for the next iteration.

Emotion Recognition Performance

  • HuBERT X-Large features + SVM classifier: 82.6% accuracy on RAVDESS
  • Fine-tuned HuBERT benchmark: 79.58% WA (speaker-dependent) on IEMOCAP
  • HuBERT-CLAP: Contrastive learning-based multimodal approach pairing HuBERT audio features with text for improved emotion recognition
  • Different transformer layers capture different emotional aspects -- middle layers often best for emotion

WavLM

WavLM (Microsoft, 2022) extends HuBERT's framework with two crucial innovations: a masked speech denoising objective and gated relative position bias in the self-attention mechanism. These changes make WavLM particularly effective for non-ASR tasks, including emotion recognition, speaker verification, and speech separation.

Architectural Innovations

1. Masked Speech Denoising

During pre-training, WavLM processes corrupted input -- speech mixed with other speakers (simulating overlap) or environmental noise -- but the target remains the clean speech pseudo-labels. This forces the model to learn:

  • Noise robustness: Representations remain stable despite acoustic interference
  • Speaker awareness: Model learns to separate and identify individual speakers
  • Paralinguistic sensitivity: Enhanced encoding of prosodic and emotional cues that persist through noise

2. Gated Relative Position Bias

Unlike Wav2Vec2 and HuBERT (which use convolutional position encoding), WavLM adds a gated relative position bias to each self-attention layer. This learnable bias improves the model's ability to capture position-dependent relationships -- particularly beneficial for content-related tasks and temporal emotion dynamics.

Why WavLM Excels at Emotion

The denoising objective endows WavLM with natural sensitivity to paralinguistic cues. By training on corrupted speech, the model learns to preserve how something is said (emotion, tone, emphasis) even when what is said is partially obscured. LoRA fine-tuning on WavLM Base+ achieves the best parameter-efficient results for emotion recognition among all SSL models tested.

Online Distillation Framework

emotion2vec builds on the data2vecA self-supervised learning framework by Meta that uses a single learning method across speech, vision, and language. It predicts contextualized latent representations rather than modality-specific targets. framework with a teacher-student architecture:

  • Initialization: Both teacher and student are initialized with the same pre-trained weights
  • Teacher network: Processes the full (unmasked) audio; parameters updated via exponential moving average (EMA)A method of updating teacher weights as a weighted average of teacher and student weights: teacher = alpha * teacher + (1-alpha) * student. This provides stable, slowly-evolving targets. of student weights
  • Student network: Processes masked audio; parameters updated via gradient descent
  • Linear projection: 384 to 768 dimensions for computing the masked language model loss

Dual-Level Loss: The Key Innovation

Utterance-Level Loss

Captures the global emotional state of the entire audio clip. Ensures the model learns holistic emotion representations -- is this clip angry, sad, or neutral overall?

Frame-Level Loss

Captures temporal emotion dynamics within the utterance. Tracks how emotion evolves across time -- the build-up to anger, the wavering in sadness, mixed emotions within a single sentence.

Why Whisper Works for Emotion

  • Massive pre-training: 680,000 hours of multilingual audio provides broad acoustic understanding
  • Paralinguistic encoding: To transcribe accurately, the encoder must implicitly model speaker characteristics, emotional states, and prosodic patterns
  • Intermediate layers: Middle encoder layers often outperform final layers for emotion -- suggesting emotion is an intermediate representation between acoustics and linguistics
  • Multilingual advantage: Unlike wav2vec2 (where English-only outperforms multilingual), Whisper benefits from multilingual pre-training for cross-lingual emotion recognition

Adaptation Strategies

  • Attention-based pooling: Multi-head Attentive Average Pooling and QKV Pooling efficiently reduce encoder output dimensionality while preserving emotional features
  • Layer selection: Best results come from intermediate transformer layers rather than the final layer
  • Knowledge distillation: Whisper encoder features can be distilled into smaller models for efficient deployment
iOS Deployment Note

WhisperKit (by Argmax) demonstrates that Whisper can run on-device on Apple Silicon with 0.46s latency and 2.2% WER. The same CoreML pipeline could be adapted for emotion recognition using the encoder representations, making Whisper a viable path for iOS deployment -- though the full Large-v3 model (1.55B params) is too large. Whisper Tiny (39M) or Small (244M) are more practical on-device candidates.

DistilHuBERT

DistilHuBERT solves the biggest practical challenge of SSL speech models: they are too large for edge deployment. By distilling a 12-layer HuBERT into a 2-layer student model, DistilHuBERT achieves a 75% compression with minimal accuracy loss -- making real-time, on-device emotion recognition feasible.

Distillation Approach

  • Student model: 2 Transformer layers (vs 12 in HuBERT-base) with the same CNN feature encoder
  • Multi-task distillation: Student hidden states are projected by linear layers to mimic hidden states from different teacher layers simultaneously
  • Layer-wise knowledge transfer: Rather than just matching the final output, the student learns to approximate multiple internal representations of the teacher

data2vec

data2vec (Meta AI, 2022) is a general-purpose self-supervised learning framework that uses the same learning method across speech, vision, and text. Its significance for emotion recognition lies primarily in providing the foundation that emotion2vec builds upon.

Core Innovation

Unlike Wav2Vec2 (which predicts quantized discrete targets) or HuBERT (which predicts cluster assignments), data2vec predicts contextualized latent representations of the full input. The teacher network produces these continuous targets from the unmasked input, and the student must predict them from the masked view.

Relevance to Emotion Recognition

  • Foundation for emotion2vec: emotion2vec directly extends data2vec with emotion-specific pre-training data and dual-level loss
  • AV-data2vec: Audio-visual extension for multimodal emotion recognition
  • Modality-agnostic learning: Demonstrates that rich representations can be learned without modality-specific targets, suggesting that the same approach could work for emotion across speech, text, and visual modalities

Linear Probing

Freeze the entire pre-trained model and train only a linear classification layer on top. This works surprisingly well when the pre-trained representations already encode emotion -- emotion2vec achieves SOTA on IEMOCAP with only linear probing. Fastest to train, lowest risk of overfitting, and demonstrates the quality of learned representations.

LoRA (Low-Rank Adaptation)

LoRALow-Rank Adaptation: Freezes pre-trained weights and injects small trainable rank decomposition matrices (A and B) into each transformer layer. The original weight matrix W is augmented by W + BA, where B and A have much smaller dimensions. freezes the pre-trained weights and injects trainable low-rank decomposition matrices into each transformer layer. Research shows LoRA achieves the best fine-tuning results on WavLM Base+ for emotion recognition, with stable and superior performance across all tested pre-trained systems while using only 2-3% of the total parameters.

Adapter Layers

Lightweight bottleneck modules inserted after the feed-forward layers in each transformer block. Only adapter weights are updated during fine-tuning, with residual connections preserving pre-trained representations. Particularly useful when you need to adapt one base model to multiple tasks (emotion + intent + speaker ID).

Full Fine-Tuning

Updating all model parameters provides the highest potential accuracy but risks catastrophic forgetting (losing general speech knowledge) and overfitting on small emotion datasets. Best reserved for scenarios with abundant labeled data and compute budget.

Practical Recommendation

Start with linear probing to benchmark representation quality. If more accuracy is needed, move to LoRA (best accuracy-to-cost ratio). Only use full fine-tuning if you have sufficient data (>10K labeled utterances) and compute resources. Combined approaches (BA + LoRA + WS + WG) can surpass full fine-tuning while using just 2-3% of parameters.

Cross-Lingual Emotion Recognition

Can a model trained on English emotional speech recognize emotion in Mandarin, Arabic, or German? Cross-lingual transfer is a critical capability for global deployment, and multilingual SSL models offer a promising path.

Key Findings

  • Whisper benefits from multilingual pre-training for cross-lingual SER -- its 680K hours of multilingual data provides broad emotional understanding
  • Wav2Vec2 English-only outperforms XLS-R (multilingual) in most languages -- a surprising result suggesting that English-only models learn more emotion-relevant representations
  • Layer selection matters: Whisper performs best with last-layer features + linear classifier; wav2vec2 benefits from middle/early layer features
  • emotion2vec generalizes across 10 languages despite being pre-trained primarily on emotion-focused data

CoreML Execution

CoreML models can run on three compute units on Apple devices:

  • CPU: Most compatible, slowest for transformer models
  • GPU: Good balance of speed and compatibility for medium models
  • Apple Neural Engine (ANE): Fastest for supported operations, but transformer self-attention may fall back to GPU/CPU. Best for the CNN feature encoder portion
Recommended iOS Strategy

For maximum quality: Fine-tune emotion2vec or WavLM on your target data, distill to a 2-4 layer student model, quantize to INT8, convert to CoreML. Target 20-50MB.

For fastest deployment: Use DistilHuBERT directly -- export to ONNX (INT8), convert to CoreML. 23MB, ready to ship.

For dual ASR + emotion: Use Whisper Tiny encoder for both transcription and emotion features. WhisperKit provides the CoreML infrastructure.

Code Example: Export Pipeline

# Step 1: Fine-tune or use pre-trained model
from transformers import HubertModel
model = HubertModel.from_pretrained("ntu-spml/distilhubert")

# Step 2: Export to ONNX
import torch
dummy_input = torch.randn(1, 16000)  # 1 second of 16kHz audio
torch.onnx.export(model, dummy_input, "distilhubert.onnx",
                   input_names=["audio"],
                   output_names=["features"],
                   dynamic_axes={"audio": {1: "seq_len"}})

# Step 3: Quantize to INT8
from onnxruntime.quantization import quantize_dynamic
quantize_dynamic("distilhubert.onnx",
                 "distilhubert_int8.onnx")

# Step 4: Convert to CoreML
import coremltools as ct
mlmodel = ct.converters.onnx.convert("distilhubert_int8.onnx")
mlmodel.save("DistilHuBERT.mlpackage")

SpeechBrain Integration

SpeechBrainAn open-source PyTorch-based speech processing toolkit. It provides modular components ("lobes") for SSL models, pre-built recipes for emotion recognition, and seamless HuggingFace integration. is the most practical way to use foundation models for emotion recognition. It wraps Wav2Vec2, HuBERT, WavLM, and Whisper as interchangeable "lobes" and provides pre-built recipes for common tasks.

Pre-Built Emotion Recognition

# Install SpeechBrain
pip install speechbrain

# Load pre-trained emotion recognition model
from speechbrain.inference.classifiers import EncoderClassifier

classifier = EncoderClassifier.from_hparams(
    source="speechbrain/emotion-recognition-wav2vec2-IEMOCAP",
    savedir="pretrained_models/emotion-recognition"
)

# Classify emotion from audio file
out_prob, score, index, text_lab = classifier.classify_file(
    "path/to/audio.wav"
)
print(f"Predicted emotion: {text_lab}")
# Output: Predicted emotion: ['ang']  (angry, hap, sad, neu)

Custom Fine-Tuning with SpeechBrain

SpeechBrain supports swapping the backbone model while keeping the same training recipe:

  • Replace wav2vec2-base with hubert-base-ls960 or wavlm-base-plus in the YAML config
  • Attentive statistical pooling converts variable-length transformer output to fixed-size embeddings
  • Additive Margin Softmax loss provides better inter-class separation than standard cross-entropy
  • Built-in data augmentation (speed perturbation, noise injection) for robust training

Getting Started Checklist

  1. Choose your deployment target (cloud, on-device, or hybrid)
  2. Start with linear probing on emotion2vec to benchmark representation quality
  3. If accuracy is insufficient, try LoRA fine-tuning on WavLM Base+
  4. For iOS: distill your best model to 2-4 transformer layers, quantize to INT8
  5. Use SpeechBrain for rapid prototyping with swappable backbones
  6. Validate cross-corpus: train on IEMOCAP, test on RAVDESS/SAVEE to check generalization

Glossary

Term Definition
SSL Self-Supervised Learning -- pre-training on unlabeled data using pretext tasks (masking, contrastive, distillation)
SER Speech Emotion Recognition -- classifying the emotional state of a speaker from audio
IEMOCAP Interactive Emotional Dyadic Motion Capture -- standard benchmark dataset (~12 hrs, 4 emotions: angry, happy, sad, neutral)
Contrastive Loss Training objective that distinguishes correct targets from distractors using cosine similarity
EMA Exponential Moving Average -- slowly updating teacher weights as a weighted combination of teacher and student weights
LoRA Low-Rank Adaptation -- parameter-efficient fine-tuning by injecting small trainable matrices into frozen layers
Linear Probing Freezing the pre-trained model and training only a linear classifier on the learned representations
WA / UA Weighted Accuracy / Unweighted Accuracy -- metrics for imbalanced emotion classification
CoreML Apple's on-device ML framework supporting CPU, GPU, and Neural Engine inference
ONNX Open Neural Network Exchange -- cross-platform model format for interoperability
Quantization Reducing numerical precision (FP32 to FP16/INT8) to shrink model size and speed up inference
Knowledge Distillation Training a smaller (student) model to replicate the behavior of a larger (teacher) model
Pretext Task An auxiliary task (masking, contrastive prediction) used during self-supervised pre-training to learn representations
PEFT Parameter-Efficient Fine-Tuning -- adapting large models by updating only a small subset of parameters (LoRA, adapters)
