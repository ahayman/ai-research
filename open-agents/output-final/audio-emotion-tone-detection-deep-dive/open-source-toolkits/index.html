Open-Source Toolkits & Libraries

The software building blocks for custom speech emotion recognition solutions — from feature extraction to deep learning frameworks, pre-trained models, and standard benchmark datasets.

Toolkit Landscape Overview

Building a speech emotion recognition (SER) system from scratch no longer requires starting from zero. A rich ecosystem of open-source tools covers every stage of the pipeline: extracting acoustic features from raw audio, training machine learning models, leveraging pre-trained deep learning weights, and evaluating against standardized benchmarks. These tools range from lightweight Python libraries to enterprise-grade C++ toolkits that can run on mobile devices.

Key Takeaways
  • openSMILE is the gold standard for handcrafted acoustic features, with native C++ that compiles for iOS/Android
  • librosa is the most accessible entry point for Python-based audio feature extraction
  • SpeechBrain provides production-ready emotion recognition recipes with pre-trained wav2vec2 models
  • emotion2vec (ACL 2024) achieves state-of-the-art results with self-supervised emotion-specific representations
  • IEMOCAP remains the benchmark dataset, but open alternatives like RAVDESS, CREMA-D, and EmoDB are more accessible
  • For an iOS pipeline, the recommended path is openSMILE C++ on-device feature extraction combined with a Core ML classifier

Toolkit Ecosystem Map

The open-source SER ecosystem can be organized by where each tool fits in the processing pipeline. Some tools (like openSMILE and librosa) focus purely on feature extraction, while others (like SpeechBrain) provide end-to-end training and inference capabilities.

openSMILE

openSMILE (open Speech and Music Interpretation by Large-space Extraction) is the most widely used toolkit for extracting handcrafted acoustic features from speech. Developed by audEERING GmbH and originally created at TU Munich, it has become the de facto standard in the Computational Paralinguistics research community.

Architecture

Written entirely in C++, openSMILE uses a modular, component-based architecture built around three core systems:

  • Component Framework — A plugin system where processing modules (sources, transforms, sinks) are dynamically configured
  • Data Memory System — Shared memory levels that allow zero-copy data exchange between components
  • Configuration Engine — .conf files that define the entire processing pipeline declaratively

Feature Sets

The eGeMAPSv02 set is the recommended starting point for most SER applications. It captures frequency parameters (F0, formants F1–F3), energy and amplitude measures, spectral features (alpha ratio, Hammarberg index, spectral slopes), and temporal features — all shown to be relevant for emotion detection in peer-reviewed research.

Feature Extraction Pipeline

Python Wrapper

The opensmile Python package wraps the C++ core, providing a clean API that returns pandas DataFramesA two-dimensional, tabular data structure in Python's pandas library. Each column is a feature, each row is an observation (one per audio file for functionals-level extraction).:

Python 
import opensmile

# Initialize with eGeMAPSv02 feature set
smile = opensmile.Smile(
    feature_set=opensmile.FeatureSet.eGeMAPSv02,
    feature_level=opensmile.FeatureLevel.Functionals,
)

# Extract features from a file → pandas DataFrame (1 x 88)
features = smile.process_file("speech.wav")

# Or from raw numpy array
import numpy as np
features = smile.process_signal(signal_array, sampling_rate=16000)

# Extract Low-Level Descriptors (frame-by-frame)
smile_lld = opensmile.Smile(
    feature_set=opensmile.FeatureSet.eGeMAPSv02,
    feature_level=opensmile.FeatureLevel.LowLevelDescriptors,
)
lld_features = smile_lld.process_file("speech.wav")
# → DataFrame with one row per frame (25 LLD features)

iOS / Mobile Integration

iOS Pipeline: openSMILE as On-Device Feature Extractor

openSMILE v3.0+ includes native iOS support via CMake. The C++ core compiles into a static library that can be linked to an Xcode project. The slim C API enables real-time feature extraction from the device microphone without network round-trips.

  • Build: CMake cross-compilation targeting iOS arm64. Produces a .a static library.
  • Integration: Call via Objective-C++ bridge or Swift C interop. Pass audio buffers from AVAudioEngine.
  • Feature Set: eGeMAPSv02 (88 features per utterance) is small enough for real-time mobile processing.
  • Classifier: Feed extracted features into a Core ML model (converted from scikit-learn SVM/MLP or PyTorch).
  • Android: Similar path via NDK/JNI. Pre-built Android integration included in v3.0+.

librosa

librosa is the most popular Python library for audio and music signal analysis. Its flat, well-documented API makes it the typical entry point for researchers building their first SER system. Current stable version is 0.11.0.

Key Features for SER

FeatureFunctionOutput ShapeSER Relevance
MFCCMel-Frequency Cepstral Coefficients. Represent the short-term power spectrum of audio on a perceptual (mel) scale. The most widely used features in speech processing. librosa.feature.mfcc() (n_mfcc, T) Primary feature; captures vocal tract shape
Mel Spectrogram librosa.feature.melspectrogram() (n_mels, T) CNN input for deep learning approaches
Chromagram librosa.feature.chroma_stft() (12, T) Pitch class; useful for tonal emotion cues
Spectral Contrast librosa.feature.spectral_contrast() (n_bands+1, T) Distinguishes harmonic vs. noisy regions
Zero Crossing Rate librosa.feature.zero_crossing_rate() (1, T) Correlates with noisiness and high-frequency content
RMS Energy librosa.feature.rms() (1, T) Loudness proxy; arousal indicator
Tonnetz librosa.feature.tonnetz() (6, T) Tonal centroid features
Python 
import librosa
import numpy as np

# Load audio (resampled to 22050 Hz by default)
y, sr = librosa.load("speech.wav", sr=16000)

# Extract features
mfcc = librosa.feature.mfcc(y=y, sr=sr, n_mfcc=13)      # (13, T)
mel = librosa.feature.melspectrogram(y=y, sr=sr)           # (128, T)
chroma = librosa.feature.chroma_stft(y=y, sr=sr)           # (12, T)

# Aggregate over time for utterance-level features
features = np.hstack([
    np.mean(mfcc, axis=1), np.std(mfcc, axis=1),
    np.mean(mel, axis=1),
    np.mean(chroma, axis=1),
])
librosa vs. openSMILE

librosa gives you raw, frame-level features and full control over aggregation. openSMILE provides expert-designed feature sets with built-in statistical functionals. For research flexibility, use librosa. For proven, standardized feature sets (especially in production), use openSMILE.

pyAudioAnalysis

pyAudioAnalysis is a Python library that bundles feature extraction with built-in classification, regression, and segmentation. It provides an all-in-one solution for audio analysis tasks.

  • 68 short-term features: MFCCs (13), chroma (12), spectral (centroid, spread, entropy, flux, rolloff), energy, ZCR, and more
  • Built-in classifiers: SVM, kNN, Random Forest, Gradient Boosting, and Extra Trees
  • Regression mode: Predict continuous values like arousal and valence scores instead of discrete emotion categories
  • Segmentation: Speaker diarization, silence removal, audio event detection
  • Visualization: Feature distribution plots, spectrograms, and audio thumbnailing

pyAudioAnalysis is best suited for rapid prototyping and small-scale projects. For production systems or cutting-edge accuracy, SpeechBrain or emotion2vec are better choices.

TorchAudio

TorchAudio is part of the PyTorch ecosystem and provides differentiable audio transforms that integrate directly into neural network training loops.

  • Differentiable transforms: Spectrogram, MelSpectrogram, MFCC, AmplitudeToDB — all as PyTorch modules
  • Pre-trained model pipelines: Wav2Vec2, HuBERT, WavLM bundles via torchaudio.pipelines
  • Audio I/O: Load and save WAV, FLAC, MP3, OGG with resampling
  • Effects: Sox-based audio effects for augmentation (pitch shift, tempo change, noise addition)
  • Kaldi compatibility: Fbank, MFCC functions matching Kaldi's output for reproducibility

TorchAudio is the natural choice when building custom PyTorch models for SER, as feature extraction becomes part of the differentiable computation graph.

senselab

senselab (from the Sensein group at MIT) is a unified Python package for biometric analysis that aggregates multiple feature extraction backends into a single API.

Multi-Backend Architecture

BackendFeaturesStrengths
OpenSMILE eGeMAPS, ComParE acoustic features Standardized paralinguistic features
Praat / Parselmouth Pitch, jitter, shimmer, formants, HNR Clinically validated voice quality measures
torchaudio Spectrogram, mel, MFCC, pitch GPU-accelerated spectral features
torchaudio-SQUIM STOI, PESQ, SI-SDR Objective audio quality metrics

Each backend can be toggled independently, and outputs are merged per audio file. This is especially useful for clinical speech analysis applications where you need features from multiple domains (phonation, articulation, prosody, spectral) in one unified feature vector.

SpeechBrain

SpeechBrain is an open-source, all-in-one conversational AI toolkit built on PyTorch. Version 1.0 (released January 2024) provides production-ready recipes for speech recognition, speaker identification, speech enhancement, and — critically for our purposes — emotion recognition.

Emotion Recognition Architecture

SpeechBrain's emotion recognition recipe uses a wav2vec2A self-supervised speech representation model from Meta (Facebook AI). Pre-trained on large amounts of unlabeled audio, it learns general-purpose speech representations that can be fine-tuned for downstream tasks like emotion recognition. encoder fine-tuned on IEMOCAP:

  • Encoder: wav2vec2-base (12 transformer layers, 768-dim hidden) pre-trained on LibriSpeech
  • Pooling: Attentive statistical pooling (learns which frames matter most for the emotion)
  • Loss: Additive Margin Softmax (AAM-Softmax) for better class separation
  • Output: 4 emotions — angry, happy (merged with excited), sad, neutral

Training Workflow

Bash 
# Clone SpeechBrain and train emotion recognition from scratch
git clone https://github.com/speechbrain/speechbrain
cd speechbrain/recipes/IEMOCAP/emotion_recognition

# Train with wav2vec2 backbone
python train_with_wav2vec2.py hparams/train_with_wav2vec2.yaml \
    --data_folder=/path/to/IEMOCAP \
    --output_folder=results/wav2vec2

# Inference with pre-trained model
python -c "
from speechbrain.pretrained.interfaces import foreign_class
classifier = foreign_class(
    source='speechbrain/emotion-recognition-wav2vec2-IEMOCAP',
    pymodule_file='custom_interface.py',
    classname='CustomEncoderWav2vec2Classifier'
)
out_prob, score, index, text_lab = classifier.classify_file('test.wav')
print(f'Emotion: {text_lab}')  # e.g., ['ang']
"

Other SpeechBrain SER Models

  • Emotion Diarization (speechbrain/emotion-diarization-wavlm-large) — WavLM-based model that assigns emotion labels over time segments, not just per utterance
  • Custom encoder support — Plug any HuggingFace transformer (Whisper, HuBERT, WavLM) as the encoder backbone
  • OpenVINO export — SpeechBrain models can be exported via ONNX for optimized inference on Intel hardware

emotion2vec

emotion2vec (ACL 2024 Findings) is the first universal speech emotion representation model. Rather than training a classifier directly, it learns emotion-specific representations through self-supervised pre-training, which can then be transferred to any downstream emotion task.

How It Works

  • Pre-training: Online distillation on 262 hours of open-source emotion data, with combined utterance-level and frame-level objectives
  • Transfer: Extract embeddings from the last layer (50Hz frame rate, stored as .npy), then train a simple linear classifier
  • Performance: State-of-the-art on IEMOCAP with just a linear probe — no fine-tuning needed
  • Cross-lingual: Tested on 10+ languages (English, Mandarin, French, German, Italian, etc.)
  • Multi-task: Works for speech emotion, song emotion, emotion in conversation, and sentiment analysis

emotion2vec vs. Fine-tuned wav2vec2

emotion2vec achieves competitive or better results than wav2vec2 fine-tuned specifically on emotion data, while being more generalizable across languages and emotion tasks. The key difference: emotion2vec is pre-trained on emotion-relevant data specifically, while wav2vec2 is pre-trained on general speech and must be fine-tuned.

Kaldi & ESPnet

Kaldi is a C++-based speech processing toolkit that has been a cornerstone of ASR research. While it does not have dedicated SER recipes, its feature extraction capabilities (MFCC, PLP, filter banks) are widely used as inputs to custom SER models. Kaldi is largely being superseded by end-to-end approaches but remains relevant for its robust, battle-tested feature extraction.

ESPnet is an end-to-end speech processing toolkit covering ASR, TTS, speech translation, enhancement, and diarization. Like Kaldi, it does not include native SER recipes, but its training infrastructure (PyTorch-based, with recipe system) can be adapted for SER by implementing custom data loaders and model architectures.

When to use Kaldi/ESPnet for SER

Only consider these toolkits if you need (a) Kaldi-compatible feature extraction for reproducibility with prior work, or (b) to embed SER within a larger speech processing pipeline (e.g., combining ASR transcription with emotion detection). For pure SER, SpeechBrain or emotion2vec are more direct paths.

Standard Benchmark Datasets

Choosing the right dataset is as important as choosing the right model. SER datasets vary enormously in size, recording conditions, emotion categories, language, and licensing. Here is a detailed look at the seven most widely used datasets.

IEMOCAP

The Interactive Emotional Dyadic Motion Capture Database from USC SAIL is the most cited SER dataset. Ten actors (5 male, 5 female) perform scripted and improvised dialogues in dyadic sessions. Approximately 12 hours of data with ~10,039 utterances annotated for 9 emotion categories plus continuous dimensions (valence, activation, dominance). The standard benchmark protocol uses four merged classes (angry, happy+excited, sad, neutral) and leave-one-session-out cross-validation.

Access: Restricted license; apply via sail.usc.edu/iemocap. Approval typically takes days to weeks.

RAVDESS

The Ryerson Audio-Visual Database of Emotional Speech and Song contains 7,356 files from 24 professional actors (12M/12F). Speech utterances cover 8 emotions (calm, happy, sad, angry, fearful, surprise, disgust, neutral) at two intensity levels (normal and strong). Audio is high-quality studio recordings.

Access: Open CC BY-NC-SA 4.0. Available on Zenodo and Kaggle.

MELD

The Multimodal EmotionLines Dataset contains 13,708 utterances from 1,433 dialogues sourced from the TV series Friends. Unique among SER datasets for its multi-party conversational context, with 7 emotion labels plus 3-class sentiment annotation. Includes audio, video, and text transcriptions.

Access: Open GPL-3.0. Available on GitHub.

CMU-MOSEI

CMU Multimodal Opinion Sentiment and Emotion Intensity is the largest multimodal emotion dataset, with 23,500+ utterances (65+ hours) from 1,000+ YouTube speakers across 250 topics. Annotations include 6 discrete emotions plus continuous dimensions. Gender-balanced and "in-the-wild" (non-acted).

Access: Research Available via CMU MultimodalSDK on GitHub.

CREMA-D

The Crowd-sourced Emotional Multimodal Actors Dataset features 7,442 utterances from 91 actors (48M/43F) spanning diverse ethnicities and ages (20–74). Six emotions at four intensity levels. The diversity of speakers makes it particularly valuable for training robust models.

Access: Open Research-friendly license. Available on GitHub.

EmoDB

The Berlin Database of Emotional Speech contains ~535 utterances from 10 German-language actors recorded in an anechoic chamber. Seven emotions: anger, boredom, disgust, fear/anxiety, happiness, sadness, neutral. Small but very high quality, extensively validated through perception tests.

Access: Open CC0-1.0 (public domain). Also available via the audb Python library.

SAVEE

The Surrey Audio-Visual Expressed Emotion database contains 480 utterances from 4 male British English actors. Seven emotions (anger, disgust, fear, happiness, sadness, surprise, neutral). Small but useful for initial experiments and combined with other datasets.

Access: Research Available from the University of Surrey.

End-to-End Custom SER System Architecture

Combining the tools described above, here is a reference architecture for building a complete SER system from open-source components. This architecture supports both a server-side training pipeline and an on-device inference path for iOS.

Recommended iOS Pipeline

For real-time emotion detection on iOS, the recommended approach combines Path A for feature extraction with a lightweight classifier:

  1. Capture audio via AVAudioEngine (16kHz, mono PCM)
  2. Extract features using openSMILE C++ (compiled for iOS arm64) with eGeMAPSv02 configuration — yields 88 features per utterance
  3. Classify using a Core ML model (converted from a scikit-learn SVM or MLP trained on the 88 features)
  4. Result: Real-time emotion labels with sub-100ms latency, no network dependency

For higher accuracy at the cost of model size, export a fine-tuned wav2vec2 model to Core ML (the base model is ~360MB, which may require distillation for mobile).

Getting Started Guide

Here are three concrete approaches for building your first SER system, ordered from simplest to most powerful.

Approach A: Traditional Features + ML Classifier

The fastest path to a working system. Use openSMILE or librosa for feature extraction and scikit-learn for classification. This approach produces lightweight models that can run on mobile devices.

Install dependencies

pip install opensmile librosa scikit-learn pandas

Download a dataset

Start with RAVDESS (open access, 1,440 utterances, 8 emotions). Download from Kaggle or Zenodo.

Extract features

Use openSMILE eGeMAPSv02 (88 features) or librosa MFCCs (13-40 coefficients). Aggregate frame-level features into utterance-level statistics.

Train a classifier

Start with SVM (RBF kernel) or MLP. Use 5-fold cross-validation with speaker-independent splits. Expect ~60-70% accuracy on RAVDESS with 8 emotions.

Evaluate

Report unweighted accuracy (UA), weighted accuracy (WA), and per-class F1 scores. Confusion matrices reveal which emotions are hardest to distinguish.

Python 
import opensmile
import pandas as pd
from sklearn.svm import SVC
from sklearn.model_selection import cross_val_score
from pathlib import Path

# 1. Extract features from all audio files
smile = opensmile.Smile(
    feature_set=opensmile.FeatureSet.eGeMAPSv02,
    feature_level=opensmile.FeatureLevel.Functionals,
)

features_list = []
labels = []
for wav_path in Path("ravdess/").glob("**/*.wav"):
    feat = smile.process_file(str(wav_path))
    features_list.append(feat)
    # Parse emotion label from RAVDESS filename
    emotion_code = wav_path.stem.split("-")[2]
    labels.append(emotion_code)

X = pd.concat(features_list).values
y = labels

# 2. Train and evaluate
clf = SVC(kernel="rbf", C=1.0, gamma="scale")
scores = cross_val_score(clf, X, y, cv=5, scoring="accuracy")
print(f"Accuracy: {scores.mean():.2f} +/- {scores.std():.2f}")

Approach B: Pre-trained Model Inference

The fastest path to high accuracy. Use a pre-trained model from HuggingFace and run inference without any training.

Python 
# Using SpeechBrain's pre-trained wav2vec2 model
from speechbrain.pretrained.interfaces import foreign_class

classifier = foreign_class(
    source="speechbrain/emotion-recognition-wav2vec2-IEMOCAP",
    pymodule_file="custom_interface.py",

)

out_prob, score, index, text_lab = classifier.classify_file("test.wav")
print(f"Predicted emotion: {text_lab}")
print(f"Confidence: {score:.3f}")

The path to maximum accuracy. Fine-tune a foundation model on your target dataset using SpeechBrain.

Which Approach Should You Choose?
  • Approach A if you need a small, interpretable model that runs on mobile/edge devices
  • Approach B if you need results quickly and your audio is similar to the training data (English, acted)
  • Approach C if you have domain-specific data and need maximum accuracy for a particular use case

Glossary

TermDefinition
SERSpeech Emotion Recognition — the task of identifying emotions from speech audio signals.
LLDLow-Level Descriptor — frame-level acoustic features (F0, energy, spectral shape) computed on short windowed segments.
FunctionalsStatistical summaries (mean, standard deviation, percentiles, regression coefficients) computed over LLDs across an entire utterance.
MFCCMel-Frequency Cepstral Coefficients — features that represent the short-term power spectrum on a perceptually motivated mel scale.
Mel SpectrogramA time-frequency representation where frequency bins are spaced according to the mel scale, mimicking human auditory perception.
eGeMAPSv02Extended Geneva Minimalistic Acoustic Parameter Set v02 — a standardized set of 88 acoustic features designed for affective computing.
ComParEComputational Paralinguistics Challenge feature set — a comprehensive set of 6,373 features used as the baseline in INTERSPEECH challenges.
wav2vec2A self-supervised speech representation model from Meta AI, pre-trained on unlabeled audio to learn general speech representations.
emotion2vecA self-supervised model pre-trained specifically on emotional speech data, yielding universal emotion representations.
Attentive PoolingA mechanism that learns to weight different frames differently when aggregating frame-level features into an utterance-level representation.
Core MLApple's framework for deploying machine learning models on iOS, macOS, watchOS, and tvOS devices.
UA (Unweighted Accuracy)The average of per-class accuracy scores, treating all classes equally regardless of their frequency in the dataset.
ArousalA continuous dimension of emotion representing the level of activation or excitement (calm to excited).
ValenceA continuous dimension of emotion representing positivity or negativity (unpleasant to pleasant).
DominanceA continuous dimension of emotion representing the degree of control or power felt by the speaker.
