Introduction: Why Use a Commercial API?

If you want to add emotion detection to an app this week, commercial APIs are the fastest path. Instead of training models, managing GPU infrastructure, and curating datasets, you send audio to an endpoint and receive emotion predictions back as JSON. The trade-offs are cost, latency, and privacy — you are sending user audio to someone else's servers.

This page evaluates every significant commercial API that can analyze emotion or sentiment from speech audio as of early 2026. We focus specifically on what matters for an iOS/Swift developer building a transcription app:

  • Detection depth — 3 sentiments vs. 48 nuanced emotions
  • Detection method — text-based sentiment vs. true acoustic/prosodic analysis
  • iOS integration ease — official Swift SDK vs. raw REST calls
  • Pricing — per-minute costs at different volumes
  • Privacy — data retention, GDPR compliance, on-premise options
  • Latency — batch vs. real-time, response times
Two Fundamentally Different Approaches

Most "sentiment from audio" APIs actually transcribe speech first, then run text sentiment analysis on the transcript. This misses vocal tone entirely. Only a few — notably Hume AI, DeepAffects, and Amazon Chime — analyze the acoustic signal directly to detect prosodic emotion cues like pitch contour, speaking rate, and vocal quality.

The API Landscape

The commercial emotion detection market segments into four tiers based on what the API actually analyzes:

Tier 1 — True Acoustic Emotion

Prosodic + Linguistic Analysis

Analyzes vocal features (pitch, rhythm, timbre) AND text. Returns granular emotion categories.

  • Hume AI (48 emotions)
  • Amazon Chime SDK (3 sentiments, hybrid)
  • DeepAffects (6 Ekman emotions)
Tier 2 — Text-Based Sentiment

Transcription + NLP Sentiment

Transcribes audio to text, then runs sentiment analysis on the transcript. Misses tone-of-voice.

  • AssemblyAI (3 sentiments)
  • Speechmatics (3 sentiments)
  • Google Cloud (2-step pipeline)
Tier 3 — Multimodal Platforms

Audio + Video + Text

Analyze multiple modalities together. Often research or enterprise focused.

  • Imentiv AI (8 emotions)
  • iMotions (research grade)
Tier 4 — Libraries / Aggregators

On-Device or Unified Access

Run locally or aggregate multiple providers behind one interface.

  • Vokaturi (5 emotions, on-device)
  • Eden AI (aggregator)

How Cloud Emotion APIs Work

All cloud-based emotion APIs follow the same general pattern: your app captures audio, sends it to a cloud endpoint, the service processes it through ML models, and returns structured emotion predictions. The critical difference is what gets analyzed.

Latency Matters

Batch APIs (upload file, wait for result) add 2-10 seconds of latency. Real-time WebSocket APIs (Hume AI, DeepAffects) can return emotion predictions within 300-500ms per utterance, enabling live emotion feedback in your app.

Hume AI

Hume AI is the standout platform for speech emotion detection, offering the richest emotion taxonomy of any commercial API. Built on peer-reviewed research, it goes far beyond simple positive/negative sentiment to measure 48 distinct emotional expressions from voice alone.

Expression Measurement API

The Expression Measurement API provides three specialized models for audio analysis:

  • Prosody ModelAnalyzes pitch contour, rhythm, tempo, loudness, and voice quality to identify emotional expression patterns in how words are spoken, independent of what words are said. — Extracts 48 emotional dimensions from the tone, rhythm, and timbre of speech. This is where the core emotion detection happens for audio.
  • Vocal Burst Model — Detects non-speech vocalizations (laughs, sighs, screams, gasps, "uh-huh", "hmm") along the same 48 dimensions. Particularly useful for capturing emotional reactions that happen between words.
  • Language Model — Analyzes the emotional content of transcribed text along 53 dimensions. Complements the prosody model by capturing what was said vs. how it was said.

Swift / iOS Integration

Hume AI is the only major emotion API with an official Swift SDK. Available via Swift Package Manager from HumeAI/hume-swift-sdk on GitHub.

  • VoiceProvider abstraction manages WebSocket connections
  • Built-in microphone capture and real-time audio playback
  • TTS file streaming support
  • Access token authentication
  • Supports iOS and macOS (including background audio mode)
Key Takeaway

Hume AI is the clear leader for iOS emotion detection. It offers the most granular emotion taxonomy (48 dimensions vs. 3-6 for competitors), true acoustic analysis (not just text sentiment), an official Swift SDK, and both batch REST and real-time WebSocket APIs. The trade-off is price: at $0.064/min it costs roughly 20x more than text-only sentiment APIs.

AssemblyAI

AssemblyAI is a leading speech-to-text platform that offers sentiment analysis as an add-on feature. It is important to understand that AssemblyAI's "sentiment analysis" works on the transcribed text, not on acoustic features. It tells you whether the words are positive, negative, or neutral — it cannot detect sarcasm, frustration, or excitement conveyed through tone of voice alone.

How It Works

  1. Submit audio via REST API
  2. Universal-2 model transcribes speech to text
  3. NLP sentiment model classifies each sentence as POSITIVE, NEGATIVE, or NEUTRAL
  4. Response includes transcript + per-sentence sentiment with confidence scores and timestamps

Capabilities

  • Sentiments: POSITIVE, NEGATIVE, NEUTRAL only
  • Granularity: Per-sentence analysis
  • Speaker Diarization: When enabled, sentiment results include speaker labels
  • Additional Features: Topic detection, entity recognition, PII redaction, content moderation
  • LLM Gateway: Apply Claude, GPT, or Gemini to transcripts for deeper analysis

iOS Integration

A community-maintained Swift package exists on the Swift Package Index. Integration uses standard REST API calls:

  • Upload audio file to AssemblyAI's servers
  • Submit transcription request with sentiment_analysis: true
  • Poll for results or use webhook
Limitation

AssemblyAI does not detect emotion from vocal tone. A person saying "I'm fine" in a shaky, tearful voice would be classified as NEUTRAL based on the text alone. For true emotion detection, combine with an acoustic analysis API like Hume AI.

Speechmatics

Speechmatics is an enterprise-grade speech-to-text platform known for supporting 50+ languages with high accuracy. Their sentiment analysis works similarly to AssemblyAI — it runs on transcribed text, not acoustic features.

Capabilities

  • Sentiments: Positive, Negative, Neutral with confidence scores
  • Modes: Batch API (primary), Real-time transcription available
  • Sentiment Language: English only (transcription supports 50+ languages)
  • Output: Labeled transcript segments with sentiment and confidence

Amazon Chime SDK

Amazon Chime SDK Call Analytics provides real-time voice tone analysis that is architecturally different from most competitors. It uses a hybrid approach: a deep neural network that jointly analyzes both acoustic/tonal features AND lexical/linguistic content. This makes it one of the few APIs that truly analyzes how something is said, not just what is said.

How It Works

  • DNN processes 5-second voice segments every 2.5 seconds
  • Jointly extracts acoustic features (pitch, energy, speaking rate) and linguistic features
  • Returns probability scores for Positive, Neutral, and Negative sentiment
  • Real-time processing designed for live call monitoring

Pricing

$0.023/min (US East region). Pay-as-you-go, no upfront commitment.

Limitations

  • Designed for call center and meeting scenarios, not general-purpose mobile apps
  • Only 3 sentiment classes (no granular emotions)
  • Requires AWS infrastructure (CloudFormation, Lambda, etc.)
  • iOS integration requires the full AWS SDK (heavy dependency)
  • Not suitable as a standalone emotion API for a transcription app

DeepAffects

DeepAffects is one of the few APIs that performs true paralinguisticRelating to the non-verbal aspects of speech communication, including tone, pitch, speaking rate, pauses, and vocal quality — everything besides the actual words being said. emotion recognition. It identifies emotions directly from acoustic properties of speech without relying on text transcription.

Capabilities

  • Emotions: 6 core Ekman emotions — Anger, Happiness, Sadness, Fear, Disgust, Surprise (plus Stress)
  • Method: Pure acoustic/paralinguistic analysis (no text dependency)
  • API Format: REST with both sync and async endpoints
  • Real-Time: Separate real-time emotion recognition API available

Imentiv AI

Imentiv AI is a multimodal emotion recognition platform that can analyze audio, video, images, and text. Their audio emotion analysis detects 8 emotions and maps them on a Valence-Arousal model.

Capabilities

  • Emotions: Happy, Sad, Angry, Fear, Surprise, Disgust, Boredom, Neutral
  • Model: Valence-Arousal mapping with real-time emotion graphing
  • Speaker Diarization: Separates and analyzes individual voices in group audio
  • Advanced: Personality trait analysis, emotion highlights, comprehensive reports

Pricing

Credit-based system. 500 free credits to start. Custom enterprise plans available.

Limitations

  • Primarily video-focused platform (audio is secondary)
  • No iOS SDK
  • Limited public API documentation

Vokaturi

Vokaturi takes a fundamentally different approach: it is a lightweight on-device library, not a cloud API. Originally an academic project, it analyzes 9 acoustic cues through a small neural network to detect 5 basic emotions. The key advantage is that no audio ever leaves the device.

Capabilities

  • Emotions: Neutral, Happiness, Sadness, Anger, Fear
  • Output: Probability percentages for each of the 5 emotions
  • Architecture: Three-level neural network with rectifying units, analyzing 9 acoustic cues
  • Processing: Entirely local (no cloud, no network required)

iOS Integration

Vokaturi is distributed as a C library. For iOS integration, you would need to create a bridging header and call the C functions from Swift. An Android JNI wrapper exists, but there is no official iOS/Swift SDK. This makes integration significantly more complex than REST-based APIs.

Privacy Advantage

Vokaturi is ideal for privacy-sensitive applications. Since it runs entirely on-device, no audio data is ever transmitted to external servers. This eliminates GDPR data transfer concerns entirely.

Google Cloud, Microsoft Azure, and Eden AI

Google Cloud

Google does not offer a dedicated speech emotion API. Achieving "emotion from audio" requires a two-step pipeline:

  1. Speech-to-Text API (Chirp 2 model) — Transcribe audio
  2. Natural Language API — Analyze sentiment on the transcript (score: -1.0 to 1.0, magnitude: 0 to infinity)

Pricing: ~$0.006-0.024/min for STT + $1.00 per 1,000 text records for NLP (first 5K free/month). The two-step approach adds latency and complexity.

Microsoft Azure (Foundry Tools)

Azure Speech Service (now under the "Foundry Tools" branding) provides comprehensive speech-to-text but no speech emotion recognition API. Azure's emotion capabilities are limited to the output side: HD Voices can detect emotional cues in text and adjust TTS speaking style accordingly. For sentiment from audio, you would need to combine Azure STT with Azure Text Analytics, similar to the Google approach.

Eden AI

Eden AI is not an emotion detection engine itself — it is an API aggregator that provides a single unified interface to multiple AI providers including emotion detection services. Key features:

  • Standardized JSON output across all underlying providers
  • Switch providers with a single parameter change
  • Pay-per-use at vendor rates (no monthly fees since January 2025)
  • GDPR engine filtering available
  • Providers include VERN AI, Google, AWS, Microsoft, and specialized engines

Hybrid Approach Recommendation

For a Swift transcription app, consider a hybrid architecture: run a lightweight on-device model (like Core ML with a SER model) for instant basic emotion feedback, then batch-send audio segments to Hume AI for detailed 48-emotion analysis when the user wants deeper insights. This gives you real-time responsiveness, offline capability, and detailed cloud-powered analysis when needed.

Practical Code Examples

Example 1: Hume AI with Official Swift SDK

Swift
import Hume

// 1. Initialize client with access token
class EmotionAnalyzer: ObservableObject {
    private var humeClient: HumeClient?
    private var voiceProvider: VoiceProvider?

    @Published var emotions: [String: Double] = [:]

    func connect() async throws {
        // Fetch access token from your backend
        let token = try await fetchAccessToken()

        humeClient = HumeClient(
            options: .accessToken(token: token)
        )

        // Create voice provider (manages WebSocket + audio)
        voiceProvider = VoiceProviderFactory
            .getVoiceProvider(client: humeClient!)

        // Connect opens WebSocket, starts microphone
        try await voiceProvider?.connect(
            configId: "your-config-id"
        )
    }

    func disconnect() {
        voiceProvider?.disconnect()
    }

    private func fetchAccessToken() async throws -> String {
        // Call your backend to get a Hume access token
        // Never embed API keys in client apps
        let url = URL(string: "https://your-backend.com/hume-token")!
        let (data, _) = try await URLSession.shared.data(from: url)
        let response = try JSONDecoder().decode(
            TokenResponse.self, from: data
        )
        return response.accessToken
    }
}

Example 2: Hume AI Expression Measurement (REST Batch)

Swift
import Foundation

struct HumeEmotionResult: Codable {
    let name: String
    let score: Double
}

struct HumePrediction: Codable {
    let emotions: [HumeEmotionResult]
    let time: TimeRange?

    struct TimeRange: Codable {
        let begin: Double
        let end: Double
    }
}

class HumeBatchAnalyzer {
    private let apiKey: String
    private let baseURL = "https://api.hume.ai/v0/batch/jobs"

    init(apiKey: String) {
        self.apiKey = apiKey
    }

    /// Submit audio file for batch emotion analysis
    func analyzeAudio(fileURL: URL) async throws -> [HumePrediction] {
        var request = URLRequest(url: URL(string: baseURL)!)
        request.httpMethod = "POST"
        request.setValue("Bearer \(apiKey)", forHTTPHeaderField: "Authorization")

        // Build multipart form data
        let boundary = UUID().uuidString
        request.setValue(
            "multipart/form-data; boundary=\(boundary)",
            forHTTPHeaderField: "Content-Type"
        )

        let audioData = try Data(contentsOf: fileURL)
        var body = Data()

        // Add models configuration
        let config = """
        {"models":{"prosody":{},"language":{},"burst":{}}}
        """
        body.appendMultipart(
            boundary: boundary, name: "json",
            contentType: "application/json",
            data: config.data(using: .utf8)!
        )

        // Add audio file
        body.appendMultipart(
            boundary: boundary, name: "file",
            filename: fileURL.lastPathComponent,
            contentType: "audio/wav",
            data: audioData
        )
        body.append("--\(boundary)--\r\n".data(using: .utf8)!)

        request.httpBody = body

        let (data, _) = try await URLSession.shared.data(for: request)
        // Parse and return predictions
        return try JSONDecoder().decode(
            [HumePrediction].self, from: data
        )
    }

    /// Get top N emotions from a prediction
    func topEmotions(
        from prediction: HumePrediction,
        count: Int = 5
    ) -> [(name: String, score: Double)] {
        prediction.emotions
            .sorted { $0.score > $1.score }
            .prefix(count)
            .map { ($0.name, $0.score) }
    }
}

Example 3: AssemblyAI Sentiment Analysis

Swift
import Foundation

struct AssemblyAITranscript: Codable {
    let id: String
    let status: String
    let text: String?
    let sentiment_analysis_results: [SentimentResult]?

    struct SentimentResult: Codable {
        let text: String
        let sentiment: String  // "POSITIVE", "NEGATIVE", "NEUTRAL"
        let confidence: Double
        let start: Int  // ms
        let end: Int    // ms
        let speaker: String?
    }
}

class AssemblyAISentimentAnalyzer {
    private let apiKey: String
    private let baseURL = "https://api.assemblyai.com/v2"

    init(apiKey: String) {
        self.apiKey = apiKey
    }

    /// Upload audio and get sentiment analysis
    func analyzeSentiment(audioURL: URL) async throws -> [AssemblyAITranscript.SentimentResult] {

        // Step 1: Upload audio file
        let uploadURL = try await uploadFile(audioURL)

        // Step 2: Create transcription with sentiment
        let transcriptId = try await createTranscript(
            audioURL: uploadURL,
            sentimentAnalysis: true
        )

        // Step 3: Poll for results
        let transcript = try await pollForResult(id: transcriptId)

        return transcript.sentiment_analysis_results ?? []
    }

    private func createTranscript(
        audioURL: String,
        sentimentAnalysis: Bool
    ) async throws -> String {
        var request = URLRequest(
            url: URL(string: "\(baseURL)/transcript")!
        )
        request.httpMethod = "POST"
        request.setValue(apiKey, forHTTPHeaderField: "Authorization")
        request.setValue("application/json", forHTTPHeaderField: "Content-Type")

        let body: [String: Any] = [
            "audio_url": audioURL,
            "sentiment_analysis": sentimentAnalysis,
            "speaker_labels": true
        ]
        request.httpBody = try JSONSerialization.data(withJSONObject: body)

        let (data, _) = try await URLSession.shared.data(for: request)
        let json = try JSONSerialization.jsonObject(with: data) as! [String: Any]
        return json["id"] as! String
    }

    private func pollForResult(
        id: String,
        interval: TimeInterval = 3.0
    ) async throws -> AssemblyAITranscript {
        while true {
            var request = URLRequest(
                url: URL(string: "\(baseURL)/transcript/\(id)")!
            )
            request.setValue(apiKey, forHTTPHeaderField: "Authorization")

            let (data, _) = try await URLSession.shared.data(for: request)
            let transcript = try JSONDecoder().decode(
                AssemblyAITranscript.self, from: data
            )

            if transcript.status == "completed" {
                return transcript
            } else if transcript.status == "error" {
                throw NSError(
                    domain: "AssemblyAI",
                    code: -1,
                    userInfo: [NSLocalizedDescriptionKey: "Transcription failed"]
                )
            }

            try await Task.sleep(nanoseconds: UInt64(interval * 1_000_000_000))
        }
    }

    private func uploadFile(_ fileURL: URL) async throws -> String {
        var request = URLRequest(
            url: URL(string: "\(baseURL)/upload")!
        )
        request.httpMethod = "POST"
        request.setValue(apiKey, forHTTPHeaderField: "Authorization")
        request.setValue(
            "application/octet-stream",
            forHTTPHeaderField: "Content-Type"
        )
        request.httpBody = try Data(contentsOf: fileURL)

        let (data, _) = try await URLSession.shared.data(for: request)
        let json = try JSONSerialization.jsonObject(with: data) as! [String: Any]
        return json["upload_url"] as! String
    }
}

Example 4: Generic REST API Pattern (DeepAffects, Imentiv, etc.)

Swift
import Foundation

/// Generic emotion API client for any REST-based provider
class GenericEmotionAPIClient {
    let baseURL: String
    let apiKey: String

    init(baseURL: String, apiKey: String) {
        self.baseURL = baseURL
        self.apiKey = apiKey
    }

    /// Send audio data and receive emotion predictions
    func analyzeEmotion(
        audioData: Data,
        endpoint: String = "/recognise_emotion",
        contentType: String = "audio/wav"
    ) async throws -> [[String: Any]] {

        let url = URL(string: "\(baseURL)\(endpoint)")!
        var request = URLRequest(url: url)
        request.httpMethod = "POST"
        request.setValue("Bearer \(apiKey)", forHTTPHeaderField: "Authorization")
        request.setValue(contentType, forHTTPHeaderField: "Content-Type")
        request.httpBody = audioData

        let (data, response) = try await URLSession.shared.data(for: request)

        guard let httpResponse = response as? HTTPURLResponse,
              (200...299).contains(httpResponse.statusCode) else {
            throw URLError(.badServerResponse)
        }

        return try JSONSerialization.jsonObject(with: data)
            as? [[String: Any]] ?? []
    }
}

This lets you swap providers without changing your UI code, A/B test accuracy, and fall back to on-device processing when offline.

Privacy and Compliance

Audio data is among the most sensitive data types. Voice recordings contain biometric voiceprints, can capture background conversations, and the emotion data itself reveals intimate psychological states. Regulatory frameworks are still catching up.

Key Regulatory Considerations

  • GDPR (EU) — Voice data is treated as sensitive personal information requiring explicit consent, secure processing, and documented retention policies
  • EU AI Act — Bans emotion recognition in workplaces and schools. This is a hard legal constraint if your app could be used in those contexts
  • CCPA (California) — Users must be able to request deletion of voice data and opt out of its sale
  • BIPA (Illinois) — Voice data may qualify as biometric data requiring written consent before collection

Privacy Best Practices for iOS Apps

  1. Minimize data sent — Send only the audio segments needed for analysis, not entire recordings
  2. Encrypt in transit — All APIs use HTTPS/WSS, but verify certificate pinning
  3. Inform users — iOS requires NSMicrophoneUsageDescription — be specific about what you analyze
  4. Offer on-device fallback — Use Core ML for basic emotion detection when privacy is paramount
  5. Request deletion — Implement API calls to request data deletion after processing
  6. Transparency — Display which emotions are being detected and why

Glossary

TermDefinition
Prosody The patterns of rhythm, stress, pitch, and intonation in speech. Prosodic features carry emotional meaning independent of the words being spoken.
Paralinguistic Non-verbal aspects of speech communication including tone, pitch, speaking rate, pauses, and vocal quality — everything besides the actual words.
Vocal Burst Non-speech vocalizations such as laughs, sighs, screams, gasps, and filler sounds ("uh-huh", "hmm") that carry emotional information.
Valence-Arousal Model A dimensional model of emotion where states are mapped on two axes: valence (pleasant to unpleasant) and arousal (calm to excited).
Ekman Emotions Six basic emotions proposed by Paul Ekman: anger, disgust, fear, happiness, sadness, and surprise. Many APIs use this as their taxonomy.
SER Speech Emotion Recognition — the task of detecting emotional states from audio speech signals using machine learning.
VAD Voice Activity Detection — identifying whether a given audio segment contains speech or silence.
ASR Automatic Speech Recognition — converting spoken audio into text (speech-to-text).
Diarization The process of segmenting an audio recording by speaker identity ("who spoke when").
WebSocket (WSS) A persistent bidirectional communication protocol enabling real-time data streaming between client and server over a single connection.
Core ML Apple's machine learning framework for running models on-device on iOS, macOS, watchOS, and tvOS with hardware acceleration.
