Metric Learning & Loss Functions for Speaker ID

How the choice of loss function during training fundamentally determines the quality of speaker embeddings -- and why getting it right is the single most impactful decision in building a speaker verification system.

Introduction

Modern speaker identification and verification systems share a common architectural pattern: an encoder network maps variable-length audio into fixed-dimensional speaker embeddings (typically 128- to 512-dimensional vectors). At test time, the system compares two embeddings using cosine similarity to decide whether they belong to the same speaker.

The encoder's architecture matters, but the loss function used to train it arguably matters more. The loss function defines what the network learns to optimize: it shapes the geometry of the embedding space, determines how tightly same-speaker utterances cluster together, how far apart different speakers are pushed, and ultimately how well the system generalizes to speakers never seen during training.

Key Insight
  • The loss function is the primary force shaping the embedding space geometry.
  • Two broad families exist: classification-based losses (Softmax, AAM-Softmax) that learn embedding-to-class-center relations, and metric learning losses (Triplet, GE2E, Prototypical) that directly learn embedding-to-embedding relations.
  • Recent hybrid approaches (Supervised Contrastive, MFCon) combine benefits of both.
  • No single loss dominates every scenario -- the best choice depends on dataset size, number of speakers, and deployment constraints.

Why Loss Functions Matter for Speaker Embeddings

Speaker verification is fundamentally an open-set problem: speakers encountered at test time are entirely different from those seen during training. This creates a critical challenge that distinguishes it from standard classification.

The Training-Testing Mismatch

During training, the system sees a finite, closed set of speakers (e.g., 7,000 speakers in VoxCeleb2). During deployment, it must verify speakers it has never seen. This mismatch means the network cannot simply memorize speaker identities -- it must learn a generalizable embedding function that captures the fundamental structure of "what makes voices different."

The loss function directly controls how well this generalization works:

  • Intra-class compactness: How tightly are embeddings from the same speaker clustered? Loose clusters lead to false rejections.
  • Inter-class separation: How far apart are embeddings from different speakers? Insufficient separation leads to false acceptances.
  • Angular uniformity: Are speakers distributed evenly on the hypersphere, or do some regions get overcrowded while others remain empty?
  • Generalization: Does the learned geometry transfer to unseen speakers, or does it overfit to the training speaker set?
Why Cosine Similarity?

Speaker embeddings live on a unit hypersphere -- the network's final layer typically applies L2 normalization. On this surface, cosine similarity equals the dot product, and the geodesic distance between two points corresponds to the angle between them. This is why angular margin methods are so natural for this domain: they directly optimize the metric used at test time.

Softmax & Its Limitations

The standard Softmax cross-entropy loss treats speaker verification as a multi-class classification problem. The network learns a weight vector for each training speaker, and the loss encourages each embedding to have maximal dot product with its corresponding class weight while minimizing dot product with all other class weights.

Lsoftmax = -log( exp(WyT x + by) / Σj exp(WjT x + bj) ) Standard Softmax Cross-Entropy Loss

How It Works

  1. The encoder produces an embedding x for each utterance.
  2. A fully-connected classification head with N output neurons (one per training speaker) computes logits.
  3. Softmax converts logits to probabilities; cross-entropy loss penalizes incorrect classifications.
  4. At test time, the classification head is discarded; only the embedding is used.

Limitations

Critical Limitations of Standard Softmax
  • No margin enforcement: Softmax only requires that the correct class logit be the largest -- it does not enforce any gap between classes. Embeddings can be separable yet geometrically close, causing verification failures.
  • Training-testing mismatch: Training optimizes a closed-set classification objective (N-way classification), but testing is an open-set verification problem (binary same/different decision). The objectives are fundamentally misaligned.
  • No intra-class compactness constraint: Softmax does not explicitly minimize the variance within each speaker's cluster.
  • Classification head dependency: The weight matrix grows linearly with the number of training speakers (N), creating scalability issues for very large datasets.

These limitations motivated two divergent lines of research: margin-enhanced softmax variants (which add geometric constraints to the classification framework) and metric learning approaches (which abandon classification entirely in favor of directly optimizing embedding geometry).

AAM-Softmax (ArcFace / CosFace)

Additive Angular Margin Softmax (AAM-Softmax), also known as ArcFace, is the dominant classification-based loss in modern speaker verification. It addresses Softmax's lack of margin by introducing an explicit angular penalty that forces embeddings away from the decision boundary.

The Evolution of Margin-Based Softmax

MethodMargin TypeFormula ModificationYear
SphereFace (A-Softmax) Multiplicative angular cos(m · θ) 2017
CosFace (AM-Softmax) Additive cosine cos(θ) - m 2018
ArcFace (AAM-Softmax) Additive angular cos(θ + m) 2019

How AAM-Softmax Works

LAAM = -log( exp(s · cos(θy + m)) / [exp(s · cos(θy + m)) + Σj≠y exp(s · cos(θj))] ) AAM-Softmax Loss: s = scale factor, m = angular margin, θ = angle between embedding and class center
  1. Normalize both the embedding x and the weight vectors W to unit length. This projects everything onto the hypersphere.
  2. Compute angles θ between the embedding and each class center via cos(θ) = WTx.
  3. Add angular margin m (typically 0.2) to the angle of the correct class. This makes it harder for the embedding to be classified correctly -- the network must push embeddings closer to their class center.
  4. Scale by factor s (typically 30) to control the sharpness of the probability distribution.
  5. Apply standard softmax cross-entropy on the modified logits.

Geometric Interpretation

The angular margin m creates a "dead zone" around each class decision boundary. An embedding must be at least m radians closer to its own class center than to any other class center to be classified correctly during training. This directly enforces inter-class angular separation on the hypersphere.

ArcFace's additive angular margin has a constant effect throughout the entire angular range (unlike SphereFace's multiplicative margin, which has non-linear effects). This provides more uniform margin enforcement and a cleaner geometric interpretation: the margin corresponds exactly to the geodesic distance on the hypersphere.

Hyperparameter Sensitivity

Margin and Scale Sensitivity

Performance of AAM-Softmax is very sensitive to the margin m and scale s parameters. Research has shown that the most common setting (AM-Softmax with m=0.3, s=30) can actually be outperformed by vanilla triplet loss if not properly tuned. Typical effective ranges: m in [0.1, 0.3], s in [15, 64].

Triplet Loss

Triplet Loss was one of the first metric learning losses applied to speaker verification, borrowed from face recognition (FaceNet, 2015). It operates on triplets of examples: an anchor, a positive (same speaker), and a negative (different speaker).

Ltriplet = max(0, ||f(a) - f(p)||2 - ||f(a) - f(n)||2 + α) Triplet Loss: a = anchor, p = positive, n = negative, α = margin

How It Works

  1. For each training step, select triplets (a, p, n) where a and p are from the same speaker and n is from a different speaker.
  2. The loss pushes the anchor-positive distance below the anchor-negative distance by at least margin α.
  3. If the constraint is already satisfied, the loss is zero (the triplet contributes no gradient).

Triplet Mining Strategies

The effectiveness of triplet loss depends critically on how triplets are selected. Most randomly chosen triplets are "easy" (the constraint is already satisfied), providing no useful gradient.

StrategySelection RuleTraining Behavior
Easy Triplets Random selection Fast but highly inefficient; most triplets have zero loss
Hard Negative Mining Select the closest negative to the anchor Fast convergence but can be unstable; may collapse embeddings
Semi-Hard Mining Negatives farther than positive but within the margin Stable compromise; the standard recommended approach
Batch Hard For each anchor, find hardest positive and hardest negative in the batch Efficient GPU utilization; every anchor contributes to the loss
Batch All Average loss over all valid triplets in the batch Comprehensive but diluted by easy triplets

A common speaker verification setup constructs mini-batches with 32 speakers, 4 utterances each, forming 128 "hard triplets" per batch via batch-hard mining.

GE2E (Generalized End-to-End) Loss

GE2E Loss, introduced by Google in 2018, was designed specifically for speaker verification. It generalizes the earlier Tuple-based End-to-End (TE2E) loss by constructing a full similarity matrix between all utterance embeddings and all speaker centroids in a batch, enabling much more efficient training.

Centroid Computation

Given a batch of N speakers with M utterances each, GE2E first computes the centroid (average embedding) for each speaker:

ck = (1/M) · Σi=1..M eki Speaker centroid: average of all M utterance embeddings for speaker k

For numerical stability and to prevent trivial solutions, GE2E uses leave-one-out centroids: when computing the similarity for utterance eki against its own speaker's centroid, that utterance is excluded from the centroid calculation.

How GE2E Training Works

  1. Batch formation: Sample N speakers (typically N=64) with M utterances each (typically M=10).
  2. Extract embeddings: Pass all N×M utterances through the encoder.
  3. Compute centroids: Calculate the mean embedding for each of the N speakers (using leave-one-out for self-comparisons).
  4. Build similarity matrix: Compute the scaled cosine similarity between every embedding and every centroid, producing an (N×M) × N matrix.
  5. Apply softmax or contrast loss: For each utterance, the loss pushes its similarity to its own speaker's centroid up, and its similarity to all other centroids down.
Sji,k = w · cos(eji, ck) + b Similarity: w and b are learnable scale and bias; cos is cosine similarity

Key Advantages Over Triplet Loss

  • All-pairs comparison: Each utterance is compared against all N centroids, not just one negative. This provides N-1 negative comparisons per sample per step.
  • Implicit hard mining: The softmax over the similarity matrix naturally emphasizes the hardest negative (the centroid most similar to the utterance from a wrong speaker).
  • No separate mining stage: Mining is implicit in the batch structure, eliminating the need for expensive pre-computation of hard triplets.
  • Efficiency: Reports show 10% EER reduction and 60% training time reduction compared to TE2E.

Angular Prototypical Loss

Angular Prototypical Loss, introduced by Chung et al. (2020) in their influential paper "In Defence of Metric Learning for Speaker Recognition," adapts prototypical networks for speaker verification with an angular (cosine-based) similarity metric. It emerged as the top-performing metric learning loss in their comprehensive comparison.

Prototypical Networks Background

Prototypical networks (Snell et al., 2017) were originally designed for few-shot classification. The key idea: compute a prototype (centroid) for each class from a small "support set," then classify query examples based on distance to each prototype.

How Angular Prototypical Loss Works

  1. Batch formation: Sample N speakers with M+1 utterances each. Reserve 1 utterance as the query; the remaining M form the support set.
  2. Compute prototypes: For each speaker, average the M support embeddings to get the prototype (centroid).
  3. Normalize: All embeddings and prototypes are L2-normalized, projecting to the unit hypersphere.
  4. Angular similarity: Compute the cosine similarity between each query and all prototypes, scaled by a learnable parameter w with bias b.
  5. Softmax loss: Apply cross-entropy loss encouraging each query to be most similar to its own prototype.
LAP = -log( exp(w · cos(qk, pk) + b) / Σj exp(w · cos(qk, pj) + b) ) Angular Prototypical Loss: q = query embedding, p = prototype centroid, w = learnable scale, b = learnable bias

Key Differences from GE2E

AspectGE2EAngular Prototypical
Query/Support split No explicit split; all utterances used Explicit 1-query, M-support split per speaker
Centroid usage Leave-one-out centroids; some centroids computed from different counts All centroids use exactly M utterances (uniform)
Test-time mimicry Partial -- similarity matrix doesn't match test protocol Strong -- support/query split directly mimics enrollment/test

Why Angular Prototypical Excels
  • Every centroid is computed from the same number of utterances, eliminating size-imbalance effects.
  • The support/query split exactly mimics the test scenario during training, removing the need for hyperparameter search on M.
  • Normalization ensures the inner product equals cosine similarity, directly optimizing the test-time metric.
  • In the Chung et al. comparison, angular prototypical outperformed Softmax, AM-Softmax, AAM-Softmax, Triplet, and GE2E.

Supervised Contrastive Loss

Supervised Contrastive (SupCon) Loss extends self-supervised contrastive learning (SimCLR, MoCo) to the supervised setting. In speaker verification, it treats all utterances from the same speaker as positives and all utterances from different speakers as negatives, creating a much richer set of training signals than triplet loss.

LSupCon = -Σi (1/|P(i)|) Σp∈P(i) log( exp(zi · zp / τ) / Σa≠i exp(zi · za / τ) ) Supervised Contrastive Loss: P(i) = set of same-speaker samples, τ = temperature parameter

How It Works

  1. Augmented pairs: For each utterance, data augmentation produces multiple views. All views of same-speaker utterances form positive pairs.
  2. All-pairs contrast: Each sample is contrasted against all other samples in the batch. Same-speaker pairs are pulled together; different-speaker pairs are pushed apart.
  3. Temperature scaling: The temperature τ controls the sharpness of the similarity distribution, analogous to the scale factor in AAM-Softmax.
  4. Multi-positive: Unlike triplet loss (one positive) or standard contrastive loss (one positive pair), SupCon leverages ALL same-speaker samples as positives simultaneously.

Angular Margin Enhancement (AMSupCon)

Recent work has enhanced supervised contrastive loss by incorporating additive angular margins into the similarity computation, combining the benefits of AAM-Softmax's margin enforcement with contrastive learning's direct embedding optimization. This "AMSupCon" approach adds a margin penalty to same-speaker similarities, forcing the network to achieve even tighter clustering to satisfy the contrastive objective.

Extends the contrastive learning paradigm to use label information. All same-speaker pairs are treated as positives, providing a much richer training signal than pairwise or triplet methods. Works particularly well when combined with angular margin variants.

Multi-Scale Feature Contrastive (MFCon) Loss

MFCon Loss, introduced in October 2024, represents the cutting edge of loss function research for speaker verification. It applies contrastive learning not just to the final speaker embedding, but to all intermediate feature maps within the network, encouraging discriminative representations at every scale of the encoder.

The Multi-Scale Insight

Modern speaker encoders (like MFA-Conformer) use multi-scale feature aggregation, extracting features at different temporal and spectral resolutions. The MFCon hypothesis is that explicitly enforcing speaker separability at each intermediate scale will produce a more robust final embedding, rather than relying on the final loss alone to propagate discriminative gradients backward through many layers.

How MFCon Works

  1. Extract multi-scale features: At each encoder block (e.g., each Conformer layer), extract the intermediate feature map and project it to a common embedding dimension.
  2. Apply contrastive loss at each scale: For each intermediate embedding, compute supervised contrastive loss -- same-speaker features should be similar, different-speaker features should be dissimilar.
  3. Combine with final embedding loss: The total loss is a weighted sum of the MFCon losses at each scale plus the primary loss (typically AAM-Softmax or AMSupCon) applied to the final embedding.
Ltotal = Lprimary(final embedding) + λ · Σl=1..L LSupCon(feature map at layer l) MFCon: Apply contrastive loss at every intermediate layer, not just the final output

Results

On the VoxCeleb1-O benchmark with an MFA-Conformer backbone:

  • MFCon alone: 4.91% relative EER reduction over the baseline AAM-Softmax
  • MFCon + AMSupCon: 9.05% relative EER reduction, demonstrating that multi-scale and final-embedding contrastive losses are complementary

The Uniformity-Alignment Framework

A useful lens for comparing losses comes from contrastive learning theory, which identifies two key properties of a good embedding space:

  • Alignment: Embeddings of the same speaker (positive pairs) should be close together. This corresponds to intra-class compactness.
  • Uniformity: The overall distribution of embeddings should be spread uniformly on the hypersphere. This maximizes the use of the available space and prevents degenerate solutions.

Classification-based losses (Softmax, AAM-Softmax) primarily optimize alignment via the classification objective, with uniformity emerging as a byproduct of multi-class competition. Contrastive losses (SupCon, MFCon) explicitly optimize both properties through their attraction/repulsion dynamics. Metric learning losses (Triplet, GE2E) focus on local alignment and relative distances, with uniformity depending on the diversity of negatives encountered during training.

Key Findings from Comparative Studies

  • Chung et al. (2020): Angular prototypical loss outperformed all classification-based (Softmax, AM-Softmax, AAM-Softmax) and metric learning (Triplet, GE2E, Prototypical) losses in a controlled comparison on VoxCeleb.
  • Margin sensitivity: Poorly tuned AM-Softmax (default m=0.3, s=30) was outperformed by vanilla triplet loss, emphasizing that hyperparameter tuning matters as much as loss function choice.
  • Batch size effect: Metric learning losses (Triplet, GE2E, Prototypical, Contrastive) benefit significantly from larger batch sizes, which provide more diverse negatives. Classification losses are less sensitive to batch size.
  • Data scale: Classification-based losses (AAM-Softmax) tend to dominate at very large training data scales (>10K speakers), while metric learning losses can be more competitive at smaller scales.

Practical Guidance: Which Loss to Use When

Production System with Large Data

Recommended: AAM-Softmax

When you have >5,000 training speakers and ample GPU resources, AAM-Softmax with a well-tuned encoder (ECAPA-TDNN, ResNet-293, or WavLM) is the proven choice. Nearly all VoxSRC challenge winners use this approach.

Tune: margin m in [0.1-0.3], scale s in [15-64]

Cutting-Edge Research

Recommended: AAM-Softmax + MFCon + AMSupCon

If squeezing the last fraction of a percent matters, combine AAM-Softmax on the final embedding with MFCon losses at intermediate layers and AMSupCon on the speaker embedding. This multi-objective approach yields the current state of the art.

Limited Training Speakers (<1K)

Recommended: Angular Prototypical Loss

With fewer training speakers, metric learning losses that directly optimize embedding relationships outperform classification approaches. Angular prototypical loss's test-time mimicry gives it an edge in low-resource scenarios.

Few-Shot Enrollment

Recommended: GE2E or Angular Prototypical

When the target system enrolls speakers with very few utterances (1-3), centroid-based losses that train with similar enrollment conditions are ideal. GE2E and prototypical networks naturally handle this setting.

On-Device / Edge Deployment

Recommended: AAM-Softmax or Triplet

Train with AAM-Softmax (best absolute performance) or triplet loss (no classification head overhead during training, simpler pipeline). The choice of loss doesn't affect inference cost -- only the embedding encoder runs at test time.

Self-Supervised / Semi-Supervised

Recommended: Supervised Contrastive + Pseudo-Labels

When labels are scarce or noisy, contrastive losses are most natural. Start with self-supervised contrastive pre-training, then fine-tune with supervised contrastive or AAM-Softmax on labeled data.

The Classification vs. Metric Learning Debate

The speaker verification community has long debated whether classification-based or metric learning losses are fundamentally superior. The evidence suggests a nuanced answer:

  • At large scale (>10K speakers, large augmentation budgets), AAM-Softmax consistently wins in practice. The VoxSRC challenge results from 2019-2023 are dominated by AAM-Softmax-trained systems.
  • In controlled comparisons (same architecture, same data, same augmentation), angular prototypical loss can match or exceed AAM-Softmax, particularly when batch sizes are large enough for effective negative sampling.
  • The gap is narrowing. Modern contrastive losses with angular margins (AMSupCon) blur the line between the two families, and hybrid approaches (AAM-Softmax + MFCon) combine their strengths.
  • Poorly tuned losses of either family will underperform well-tuned losses of the other. Hyperparameter optimization often matters more than the choice of loss function itself.
Bottom Line
  • For most practitioners, AAM-Softmax with careful margin/scale tuning is the safe, high-performing default.
  • For researchers pushing the frontier, combining AAM-Softmax with contrastive auxiliary losses (MFCon, AMSupCon) yields measurable gains.
  • For low-resource or few-shot scenarios, angular prototypical loss is the strongest metric learning choice.
  • Never use vanilla Softmax for production speaker verification -- always add margins.
