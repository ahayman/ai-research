Coordinator Pattern in SwiftUI

Hierarchical navigation management · iOS 16+ · Advanced

Overview

The Coordinator Pattern separates navigation logic from view logic by introducing dedicated coordinator objects that manage entire navigation flows. Each coordinator owns a NavigationPath (or sheet bindings) and decides which screen to show next. Views become simple, reusable UI components with zero knowledge of where they sit in the app's navigation hierarchy.

Key Idea

Views don't navigate. Coordinators navigate. Views request actions (e.g., "user tapped Settings"), and the coordinator decides what happens next (push SettingsView, show a sheet, or start a child coordinator flow).

Origins & Motivation

The Coordinator Pattern was popularized by Soroush Khanlou in 2015 for UIKit, where view controllers were notoriously tangled with navigation logic. The same problems exist in SwiftUI when navigation grows beyond a few screens:

  • Scattered navigation: NavigationLink destinations spread across dozens of views
  • Tight coupling: ViewA imports ViewB just to push it — changing one breaks the other
  • Untestable flows: You can't test "after login, show the home screen" without rendering UI
  • Reuse blocked: A ProductDetailView that always pushes ReviewView can't be used in a different flow
  • Deep linking pain: Constructing a navigation path from a URL means touching every view in the chain

Coordinators solve all of these by centralizing navigation decisions in dedicated objects that sit above the view layer.

Coordinator Protocol

Start with a base protocol that all coordinators conform to:

// CoordinatorProtocol.swift
import SwiftUI

@MainActor
protocol Coordinator: AnyObject, Observable {
    associatedtype ContentView: View

    /// Build the coordinator's root view
    @ViewBuilder func start() -> ContentView
}

/// For coordinators that manage a NavigationStack
@MainActor
protocol StackCoordinator: Coordinator {
    associatedtype Route: Hashable

    var path: NavigationPath { get set }

    func push(_ route: Route)
    func pop()
    func popToRoot()

    @ViewBuilder func destination(for route: Route) -> some View
}

extension StackCoordinator {
    func push(_ route: Route) { path.append(route) }
    func pop() { guard !path.isEmpty else { return }; path.removeLast() }
    func popToRoot() { path = NavigationPath() }
}

Why a Protocol?

The protocol gives you a consistent interface across all coordinators. You can swap coordinator implementations for testing, compose them, and enforce that every coordinator provides a start() view and route-to-view mapping.

App Coordinator (Root)

The AppCoordinator is the root of the hierarchy. For a tabbed app, it owns the TabView and creates child coordinators for each tab:

// AppCoordinator.swift
import SwiftUI

@Observable
@MainActor
final class AppCoordinator: Coordinator {
    enum Tab: Hashable {
        case home, settings, profile
    }

    var selectedTab: Tab = .home

    // Child coordinators — one per tab
    let homeCoordinator = HomeCoordinator()
    let settingsCoordinator = SettingsCoordinator()
    let profileCoordinator = ProfileCoordinator()

    @ViewBuilder
    func start() -> some View {
        TabView(selection: $selectedTab) {
            homeCoordinator.start()
                .tabItem { Label("Home", systemImage: "house") }
                .tag(Tab.home)

            settingsCoordinator.start()
                .tabItem { Label("Settings", systemImage: "gear") }
                .tag(Tab.settings)

            profileCoordinator.start()
                .tabItem { Label("Profile", systemImage: "person") }
                .tag(Tab.profile)
        }
    }

    /// Pop-to-root on tab reselection
    func selectTab(_ tab: Tab) {
        if selectedTab == tab {
            switch tab {
            case .home: homeCoordinator.popToRoot()
            case .settings: settingsCoordinator.popToRoot()
            case .profile: profileCoordinator.popToRoot()
            }
        }
        selectedTab = tab
    }

    /// Deep link handling
    func handleDeepLink(url: URL) {
        guard let components = URLComponents(url: url, resolvingAgainstBaseURL: false) else { return }
        let pathParts = components.path.split(separator: "/")

        switch pathParts.first {
        case "home":
            selectedTab = .home
            homeCoordinator.handleDeepLink(pathParts: Array(pathParts.dropFirst()))
        case "settings":
            selectedTab = .settings
            settingsCoordinator.handleDeepLink(pathParts: Array(pathParts.dropFirst()))
        case "profile":
            selectedTab = .profile
            profileCoordinator.handleDeepLink(pathParts: Array(pathParts.dropFirst()))
        default:
            break
        }
    }
}

// App entry point
@main
struct MyApp: App {
    @State private var appCoordinator = AppCoordinator()

    var body: some Scene {
        WindowGroup {
            appCoordinator.start()
                .onOpenURL { url in
                    appCoordinator.handleDeepLink(url: url)
                }
        }
    }
}

Child Coordinators

Each tab has its own coordinator that manages a NavigationStack and defines routes:

// HomeCoordinator.swift
import SwiftUI

@Observable
@MainActor
final class HomeCoordinator: StackCoordinator {
    enum Route: Hashable {
        case detail(itemId: String)
        case category(name: String)
        case favorites
    }

    var path = NavigationPath()

    // Sheet state for child coordinators
    var showAuth = false
    var authCoordinator: AuthCoordinator?

    @ViewBuilder
    func start() -> some View {
        NavigationStack(path: $path) {
            HomeScreen(coordinator: self)
                .navigationDestination(for: Route.self) { route in
                    self.destination(for: route)
                }
        }
        .sheet(isPresented: $showAuth) {
            if let auth = authCoordinator {
                auth.start()
            }
        }
    }

    @ViewBuilder
    func destination(for route: Route) -> some View {
        switch route {
        case .detail(let itemId):
            DetailScreen(itemId: itemId, coordinator: self)
        case .category(let name):
            CategoryScreen(name: name, coordinator: self)
        case .favorites:
            FavoritesScreen(coordinator: self)
        }
    }

    // Actions called by views
    func showDetail(itemId: String) {
        push(.detail(itemId: itemId))
    }

    func showCategory(name: String) {
        push(.category(name: name))
    }

    func showFavorites() {
        push(.favorites)
    }

    func startAuthFlow() {
        authCoordinator = AuthCoordinator(delegate: self)
        showAuth = true
    }

    func handleDeepLink(pathParts: [Substring]) {
        popToRoot()
        guard let first = pathParts.first else { return }
        switch first {
        case "detail":
            if let id = pathParts.dropFirst().first {
                push(.detail(itemId: String(id)))
            }
        case "favorites":
            push(.favorites)
        default: break
        }
    }
}

// The delegate pattern for child → parent communication
extension HomeCoordinator: AuthCoordinatorDelegate {
    func authDidComplete(user: User) {
        showAuth = false
        authCoordinator = nil
        // Navigate to user's profile or refresh
    }

    func authDidCancel() {
        showAuth = false
        authCoordinator = nil
    }
}

Auth Coordinator (Sub-flow)

        case register
        case forgotPassword

        self.delegate = delegate

        case .register:

        case .forgotPassword:

        }

    func showRegister() { push(.register) }
    func showForgotPassword() { push(.forgotPassword) }

    func completeLogin(user: User) {
        delegate?.authDidComplete(user: user)
    }

    func cancel() {
        delegate?.authDidCancel()
    }
}

Modal Handling

Coordinators manage modals (sheets, fullScreenCovers, alerts) through state properties:

@Observable
@MainActor
final class SettingsCoordinator: StackCoordinator {
    enum Route: Hashable {
        case notifications
        case account
        case appearance
    }

    enum SheetType: Identifiable {
        case feedback
        case about

        var id: String { String(describing: self) }
    }

    enum AlertType: Identifiable {
        case confirmLogout
        case confirmDeleteAccount

        var id: String { String(describing: self) }
    }

    var path = NavigationPath()
    var activeSheet: SheetType?
    var activeAlert: AlertType?

    @ViewBuilder
    func start() -> some View {
        NavigationStack(path: $path) {
            SettingsScreen(coordinator: self)
                .navigationDestination(for: Route.self) { route in
                    self.destination(for: route)
                }
        }
        .sheet(item: $activeSheet) { sheet in
            switch sheet {
            case .feedback: FeedbackView()
            case .about: AboutView()
            }
        }
        .alert(item: $activeAlert) { alert in
            switch alert {
            case .confirmLogout:
                Alert(
                    title: Text("Log Out?"),
                    message: Text("Are you sure you want to log out?"),
                    primaryButton: .destructive(Text("Log Out")) { self.performLogout() },
                    secondaryButton: .cancel()
                )
            case .confirmDeleteAccount:
                Alert(
                    title: Text("Delete Account?"),
                    message: Text("This action cannot be undone."),
                    primaryButton: .destructive(Text("Delete")) { self.performDeleteAccount() },
                    secondaryButton: .cancel()
                )
            }
        }
    }

    @ViewBuilder
    func destination(for route: Route) -> some View {
        switch route {
        case .notifications: NotificationsScreen(coordinator: self)
        case .account: AccountScreen(coordinator: self)
        case .appearance: AppearanceScreen()
        }
    }

    func showNotifications() { push(.notifications) }
    func showAccount() { push(.account) }
    func showAppearance() { push(.appearance) }
    func showFeedback() { activeSheet = .feedback }
    func showAbout() { activeSheet = .about }
    func confirmLogout() { activeAlert = .confirmLogout }
    func confirmDeleteAccount() { activeAlert = .confirmDeleteAccount }

    private func performLogout() { /* ... */ }
    private func performDeleteAccount() { /* ... */ }

    func handleDeepLink(pathParts: [Substring]) {
        popToRoot()
        guard let first = pathParts.first else { return }
        switch first {
        case "notifications": push(.notifications)
        case "account": push(.account)
        default: break
        }
    }
}

@Observable vs ObservableObject

iOS 17 introduced the @Observable macro, which is the modern approach. For iOS 16 support, use ObservableObject:

Feature@Observable (iOS 17+)ObservableObject (iOS 16)
Declaration@Observable classclass: ObservableObject
PropertiesAll tracked automaticallyNeed @Published
Injection@Environment@EnvironmentObject
View updatesOnly for accessed propertiesAll @Published changes
PerformanceBetter (fine-grained)Can over-render
Opt-out@ObservationIgnoredDon't use @Published

// For iOS 16 compatibility
final class HomeCoordinatorLegacy: ObservableObject, StackCoordinator {
    enum Route: Hashable {
        case detail(itemId: String)
        case favorites
    }

    @Published var path = NavigationPath()
    @Published var showAuth = false

    @ViewBuilder
    func start() -> some View {
        NavigationStack(path: $path) {
            HomeScreen(coordinator: self)
                .navigationDestination(for: Route.self) { route in
                    self.destination(for: route)
                }
        }
    }

    // ... same methods as before
}

// Usage in views — @EnvironmentObject instead of @Environment
struct HomeScreen: View {
    @EnvironmentObject var coordinator: HomeCoordinatorLegacy

    var body: some View {
        List {
            Button("Show Detail") { coordinator.showDetail(itemId: "1") }
        }
    }
}

Dependency Injection

Coordinators are ideal injection points for dependencies. They create views and can pass services:

@Observable
@MainActor
final class HomeCoordinator: StackCoordinator {
    // Dependencies injected at init
    let productService: ProductServiceProtocol
    let analyticsService: AnalyticsProtocol
    let userSession: UserSession

    init(
        productService: ProductServiceProtocol,
        analyticsService: AnalyticsProtocol,
        userSession: UserSession
    ) {
        self.productService = productService
        self.analyticsService = analyticsService
        self.userSession = userSession
    }

    @ViewBuilder
    func destination(for route: Route) -> some View {
        switch route {
        case .detail(let id):
            // Coordinator passes only what the view needs
            DetailScreen(
                viewModel: DetailViewModel(
                    itemId: id,
                    productService: productService,
                    analytics: analyticsService
                ),
                onBack: { self.pop() }
            )
        case .favorites:
            FavoritesScreen(
                viewModel: FavoritesViewModel(
                    userSession: userSession,
                    productService: productService
                )
            )
        }
    }
}

Testing Navigation Logic

The biggest advantage of coordinators: you can unit test navigation without rendering views:

import XCTest
@testable import MyApp

@MainActor
final class HomeCoordinatorTests: XCTestCase {

    var coordinator: HomeCoordinator!

    override func setUp() {
        coordinator = HomeCoordinator(
            productService: MockProductService(),
            analyticsService: MockAnalytics(),
            userSession: MockUserSession()
        )
    }

    func testShowDetailPushesRoute() {
        coordinator.showDetail(itemId: "42")
        XCTAssertEqual(coordinator.path.count, 1)
    }

    func testPopRemovesLastRoute() {
        coordinator.showDetail(itemId: "1")
        coordinator.showFavorites()
        XCTAssertEqual(coordinator.path.count, 2)

        coordinator.pop()
        XCTAssertEqual(coordinator.path.count, 1)
    }

    func testPopToRootClearsPath() {
        coordinator.showDetail(itemId: "1")
        coordinator.showCategory(name: "Electronics")
        coordinator.showFavorites()
        XCTAssertEqual(coordinator.path.count, 3)

        coordinator.popToRoot()
        XCTAssertTrue(coordinator.path.isEmpty)
    }

    func testStartAuthFlowPresentsSheet() {
        XCTAssertFalse(coordinator.showAuth)
        XCTAssertNil(coordinator.authCoordinator)

        coordinator.startAuthFlow()

        XCTAssertTrue(coordinator.showAuth)
        XCTAssertNotNil(coordinator.authCoordinator)
    }

    func testAuthCompleteDismissesSheet() {
        coordinator.startAuthFlow()
        let mockUser = User(id: "1", name: "Test")

        coordinator.authDidComplete(user: mockUser)

        XCTAssertFalse(coordinator.showAuth)
        XCTAssertNil(coordinator.authCoordinator)
    }

    func testDeepLinkNavigatesToDetail() {
        coordinator.handleDeepLink(pathParts: ["detail", "99"])
        XCTAssertEqual(coordinator.path.count, 1)
    }

    func testDeepLinkResetsExistingPath() {
        coordinator.showFavorites()
        coordinator.showDetail(itemId: "1")
        XCTAssertEqual(coordinator.path.count, 2)

        coordinator.handleDeepLink(pathParts: ["favorites"])
        XCTAssertEqual(coordinator.path.count, 1)
    }
}

Zero UI Required

These tests run in milliseconds with no simulator needed. You're testing that tapping "Show Detail" pushes a route, that deep links construct the right path, and that auth flow lifecycle works correctly — all as pure logic.

Complete Mini App

Here's a copy-paste-ready example you can drop into Xcode. It demonstrates AppCoordinator with two tabs, a child AuthCoordinator presented as a sheet, and full deep linking:

// CoordinatorDemoApp.swift — Complete working app
import SwiftUI

// MARK: - Models
struct User: Hashable, Identifiable {
    let id: String
    let name: String
}

struct Product: Hashable, Identifiable {
    let id: String
    let name: String
    let price: Double
}

// MARK: - Coordinator Protocol
@MainActor
protocol AppStackCoordinator: AnyObject, Observable {
    associatedtype Route: Hashable
    var path: NavigationPath { get set }
}

extension AppStackCoordinator {
    func push(_ route: Route) { path.append(route) }
    func pop() { guard !path.isEmpty else { return }; path.removeLast() }
    func popToRoot() { path = NavigationPath() }
}

// MARK: - Auth Coordinator
@MainActor protocol AuthDelegate: AnyObject {
    func didLogin(user: User)
    func didCancelAuth()
}

@Observable @MainActor
final class AuthCoordinator: AppStackCoordinator {
    enum Route: Hashable { case register }
    var path = NavigationPath()
    weak var delegate: AuthDelegate?

    init(delegate: AuthDelegate) { self.delegate = delegate }

    @ViewBuilder func start() -> some View {
        NavigationStack(path: $path) {
            VStack(spacing: 20) {
                Text("Login").font(.largeTitle.bold())
                Button("Log In") {
                    self.delegate?.didLogin(user: User(id: "1", name: "Aaron"))
                }
                .buttonStyle(.borderedProminent)
                Button("Create Account") { self.push(.register) }
                Button("Cancel") { self.delegate?.didCancelAuth() }
                    .foregroundStyle(.secondary)
            }
            .navigationDestination(for: Route.self) { route in
                switch route {
                case .register:
                    VStack(spacing: 20) {
                        Text("Register").font(.largeTitle.bold())
                        Button("Complete Registration") {
                            self.delegate?.didLogin(user: User(id: "2", name: "New User"))
                        }
                        .buttonStyle(.borderedProminent)
                    }
                }
            }
        }
    }
}

// MARK: - Home Coordinator
@Observable @MainActor
final class HomeCoordinator: AppStackCoordinator, AuthDelegate {
    enum Route: Hashable {
        case productDetail(Product)
        case favorites
    }

    var path = NavigationPath()
    var showAuth = false
    var authCoordinator: AuthCoordinator?
    var currentUser: User?

    let products = [
        Product(id: "1", name: "MacBook Pro", price: 1999),
        Product(id: "2", name: "iPhone 16", price: 999),
        Product(id: "3", name: "iPad Air", price: 599),
    ]

    @ViewBuilder func start() -> some View {
        NavigationStack(path: $path) {
            List {
                if let user = currentUser {
                    Section { Text("Welcome, \(user.name)!") }
                } else {
                    Section {
                        Button("Log In") { self.startAuth() }
                    }
                }

                Section("Products") {
                    ForEach(products) { product in
                        Button {
                            self.push(.productDetail(product))
                        } label: {
                            HStack {
                                Text(product.name)
                                Spacer()
                                Text("$\(product.price, specifier: "%.0f")")
                                    .foregroundStyle(.secondary)
                            }
                        }
                    }
                }

                Section {
                    Button("Favorites") { self.push(.favorites) }
                }
            }
            .navigationTitle("Home")
            .navigationDestination(for: Route.self) { route in
                switch route {
                case .productDetail(let product):
                    VStack(spacing: 16) {
                        Text(product.name).font(.largeTitle.bold())
                        Text("$\(product.price, specifier: "%.2f")").font(.title2)
                        Button("Buy Now") { /* purchase logic */ }
                            .buttonStyle(.borderedProminent)
                    }
                    .navigationTitle(product.name)
                case .favorites:
                    Text("Your Favorites")
                        .font(.largeTitle.bold())
                        .navigationTitle("Favorites")
                }
            }
        }
        .sheet(isPresented: $showAuth) {
            authCoordinator?.start()
        }
    }

    func startAuth() {
        authCoordinator = AuthCoordinator(delegate: self)
        showAuth = true
    }

    // AuthDelegate
    func didLogin(user: User) {
        currentUser = user
        showAuth = false
        authCoordinator = nil
    }

    func didCancelAuth() {
        showAuth = false
        authCoordinator = nil
    }

    func handleDeepLink(_ parts: [Substring]) {
        popToRoot()
        guard let first = parts.first else { return }
        if first == "product", let id = parts.dropFirst().first,
           let product = products.first(where: { $0.id == String(id) }) {
            push(.productDetail(product))
        } else if first == "favorites" {
            push(.favorites)
        }
    }
}

        popToRoot()
        if parts.first == "notifications" { push(.notifications) }
    }
}

// MARK: - App Coordinator
@Observable @MainActor
final class AppCoordinator {
    enum Tab: Hashable { case home, settings }

    var selectedTab: Tab = .home
    let home = HomeCoordinator()
    let settings = SettingsCoordinator()

    @ViewBuilder func start() -> some View {
        TabView(selection: Binding(
            get: { self.selectedTab },
            set: { newTab in
                if newTab == self.selectedTab {
                    switch newTab {
                    case .home: self.home.popToRoot()
                    case .settings: self.settings.popToRoot()
                    }
                }
                self.selectedTab = newTab
            }
        )) {
            home.start()
                .tabItem { Label("Home", systemImage: "house") }
                .tag(Tab.home)

            settings.start()
                .tabItem { Label("Settings", systemImage: "gear") }
                .tag(Tab.settings)
        }
    }

    func handleURL(_ url: URL) {
        let parts = url.pathComponents.dropFirst().map { Substring($0) }
        guard let first = parts.first else { return }
        switch first {
        case "home":
            selectedTab = .home
            home.handleDeepLink(Array(parts.dropFirst()))
        case "settings":
            selectedTab = .settings
            settings.handleDeepLink(Array(parts.dropFirst()))
        default: break
        }
    }
}

            coordinator.start()

Advantages

  • Complete separation of navigation from views
  • Unit test all navigation flows without UI
  • Views are fully reusable across different flows
  • Deep linking is hierarchical and clean
  • Teams can work on features independently
  • Sub-flows (auth, onboarding) are self-contained
  • Consistent pattern from UIKit — familiar to many

Disadvantages

  • Significant boilerplate vs. simple patterns
  • Overkill for small apps (< 10 screens)
  • SwiftUI's declarative model makes it less natural than UIKit
  • Coordinator lifecycle management adds complexity
  • Delegate protocols add indirection
  • New team members need onboarding
  • Some fight against SwiftUI's built-in navigation model

When to Use This Approach

Good Fit

  • Apps with 15+ screens and complex navigation graphs
  • Teams of 3+ developers working on different features
  • Apps with distinct sub-flows (auth, onboarding, checkout)
  • Projects that require comprehensive navigation testing
  • Apps migrating from UIKit that already use coordinators
  • Enterprise apps with strict architecture requirements

Consider Alternatives

  • Simple apps with < 10 screens — use Enum Router instead
  • Solo developers on small projects — overhead may not pay off
  • Prototypes and MVPs — keep it simple, refactor later
  • If the team prefers TCA — use TCA's built-in navigation instead

Coordinator vs. Router Pattern

The main distinction:

  • Router: A single object managing routes for one NavigationStack. Good for most apps.
  • Coordinator: A hierarchical system where coordinators create child coordinators for sub-flows. Scales to very large apps.

In practice, a coordinator contains a router. The coordinator adds lifecycle management, parent-child relationships, and delegate-based communication between flows. If you only need centralized routing without sub-flows, the Enum Router is simpler and sufficient.
