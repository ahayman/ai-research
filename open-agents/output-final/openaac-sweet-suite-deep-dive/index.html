Chapter 1

From a Father's Frustration to Open Source

The story of Sweet Suite begins not with a business plan but with a parent's desperation -- and grew into a movement that redefined openness in the AAC industry.

2014
CoughDrop Founded
Brian Whitmer, a software developer searching for an AAC solution for his daughter with Rett Syndrome, collaborates with 30+ SLPs, OTs, and AT specialists to build CoughDrop -- a cloud-based, cross-platform AAC app that brought modern web technology to a field dominated by legacy software.
January 2015
Open Board Format Launched
Whitmer initiates the Open Board Format (OBF) project, creating the first vendor-neutral file format for AAC communication boards. This enables boards to be exported and imported across different AAC applications -- a groundbreaking step toward interoperability.
2016
CoughDrop Goes Open Source
The entire CoughDrop codebase is released as open source under AGPLv3, making it one of the first full-featured AAC platforms to do so. The full production codebase -- the same code running the commercial service -- is published for anyone to inspect, modify, and deploy.
2018-2022
OpenAAC Ecosystem Grows
OpenAAC is established as a 501(c)(3) non-profit. The ecosystem expands with OpenSymbols (60,000+ symbols), AAC Shim (web accessibility for AAC users), AAC in the Cloud conference, the knAAC Awards, and AAC Metrics (board set analysis tools).
Early 2023
Forbes AAC Acquires CoughDrop
Forbes AAC (makers of ProSlate and WinSlate devices) acquires CoughDrop. Commercial assets transfer to MavWare, LLC. The open-source codebase continues under the OpenAAC umbrella.
2023-2024
Rebranded as Sweet Suite AAC
The open-source repository is rebranded from CoughDrop to "Sweet Suite AAC" under the open-aac GitHub organization, maintaining the same Rails + Ember architecture while establishing a distinct identity from the proprietary CoughDrop commercial service.
2025-2026
Active Development Continues
Sweet Suite repository updated April 2025 (6,111 commits). OBF gem and OpenAAC.org remain actively maintained. The 2026 AAC in the Cloud conference is scheduled for June 24-25 with theme "Choices Give us a Voice."

The philosophy was clear: communication is a human right, and the tools that enable communication should not be locked behind proprietary walls. The commercial entity funds ongoing development, while the open-source fork ensures the technology remains available to the community regardless of corporate decisions.

Chapter 3

Core Features

Sweet Suite provides a comprehensive AAC experience with communication boards, cloud sync, supervisor tools, and robust accessibility -- all from a single open-source codebase.

💬

Communication Boards

Customizable grid-based boards with symbol-based pictographic buttons, labels, and actions. Boards link hierarchically, supporting core vocabulary and topic-based organization. All boards use the Open Board Format for cross-app portability.

🔊

Speech Output

Text-to-speech via the Web Speech API with support for multiple voices and languages. Works offline once cached, ensuring a communicator never loses their voice due to connectivity issues.

☁️

Cloud Sync & Offline

Cloud-based architecture syncs changes across all devices automatically. Board data is cached via IndexedDB, assets via Application Cache. Works fully offline and syncs when reconnected.

👥

Supervisor System

Free supervisor accounts for therapists, parents, and teachers. Remote monitoring, language modeling, board modification, usage reports, and secure team messaging -- all from the supervisor's own device.

📊

Usage Tracking

Optional tracking of all button presses in Speak Mode. Data backed up to cloud for remote review by communicators and linked supervisors. Helps track language development over time.

♿

20+ Access Methods

Direct touch, gaze tracking, head tracking (via cordova_face), switch scanning, pointer/mouse, and keyboard navigation. Designed for users with complex physical needs across all motor ability levels.

Chapter 4

The OpenAAC Ecosystem

Sweet Suite is the flagship application of a broader constellation of standards, tools, libraries, and community initiatives -- all governed by a 501(c)(3) non-profit.

📋
Open Board Format (OBF/OBZ)
Vendor-neutral JSON/ZIP format for AAC communication boards. The de facto interoperability standard supported by Sweet Suite, Cboard, AsTeRICS Grid, PiCom, and others.
Standard 7 Stars MIT
🎨
OpenSymbols
Aggregates 60,000+ symbols from ARASAAC, Sclera, Mulberry, OpenMoji, and Global Symbols into a single searchable API. All open-licensed.
Library 21 Stars 60K+ Symbols
🌐
AAC Shim
JavaScript utility enabling AAC systems to interact with regular websites. Removes barriers to internet access for AAC users who cannot type or speak.
Tool JavaScript Web A11y
📏
AAC Metrics
Tool for objectively analyzing and comparing grid-based AAC board sets. Enables evidence-based evaluation of vocabulary coverage and navigation efficiency.
Analysis 3 Stars MIT
🎬
AAC in the Cloud
Annual free, fully-online conference for the global AAC community. 2026 edition themed "Choices Give us a Voice" on June 24-25. Supports CEU credit tracking.
Conference Free Annual

knAAC Awards
Annual awards celebrating excellence in AAC across 18 categories. 2024 edition recognized 100+ nominees. Honors split between "Keepers" and "Noteworthies."

OBF Ruby Gem
Library for programmatically parsing and generating OBF/OBZ files. Essential developer tool for building OBF-compatible applications.

cordova_face
Mobile face and head tracking with Cordova support. Enables head-based access methods for AAC users who cannot use touch or gaze tracking.

Chapter 6

AAC Industry GPA (2024)

OpenAAC's annual ecosystem assessment grades the entire AAC field across 10 dimensions. Six expert evaluators complete Likert-scale surveys, and the results are published publicly. The 2024 overall GPA: 1.9/4.0 (C-).

AAC Ecosystem GPA by Category
Source: OpenAAC 2024 Ecosystem Assessment -- Six expert evaluators from diverse AAC backgrounds

Education earned an F (0.6/4.0) -- reflecting persistent problems with literacy instruction for AAC users, device restrictions in schools, and outdated educational practices. Meanwhile, Families earned a B (3.0), the strongest area, showing improved resources and self-organizing support groups. The overall 1.9 GPA reveals an industry with significant systemic issues.

Chapter 8

Strengths & Weaknesses

An honest assessment of where Sweet Suite excels and where it faces challenges.

Strengths
+ Fully Open Source (AGPLv3) -- Complete codebase available for inspection, modification, self-hosting
+ Battle-Tested Maturity -- 6,111 commits over a decade; same code powers CoughDrop commercially
+ Standards Leader -- Created OBF, the de facto interoperability standard for AAC boards
+ Comprehensive Features -- Supervisor system, usage tracking, cloud sync, offline support
+ Non-Profit Governance -- 501(c)(3) ensures community benefit over profit
+ Ecosystem Approach -- Not just an app: standards, symbols, analysis tools, conference, awards
+ 20+ Access Methods -- Gaze, head tracking, scanning, touch, keyboard, pointer
+ Cross-Platform -- Web, iOS, Android, Windows, Mac, ChromeOS from one codebase
+ i18n First -- Multi-locale support built in from the ground up
Weaknesses
- Complex Self-Hosting -- Rails, PostgreSQL, Redis, Node.js, ImageMagick, AWS, Google APIs all required
- Small External Community -- Only 5 stars / 5 forks despite 6,111 commits
- Undocumented API -- REST API is "open and completely-undocumented"
- Aging Frontend Stack -- Ember.js + Bower (deprecated since 2017) hinders contributor onboarding
- Identity Confusion -- Sweet Suite vs. CoughDrop vs. OpenAAC vs. Forbes AAC relationships unclear
- Heavy Cloud Dependencies -- AWS + Google APIs required even for self-hosted instances
- Limited Documentation -- Setup and configuration guides are sparse
- Open Integrations: F -- Third-party integration remains practically nonexistent
- Open Exports: D -- Vocabulary portability still a challenge despite OBF existing

Chapter 10

The Road Ahead

Challenges and opportunities for Sweet Suite and the OpenAAC ecosystem.

Opportunities

AI Integration: Large language models and speech AI could bring predictive text, context-aware suggestions, and natural-sounding synthesis to AAC. Sweet Suite's open architecture is well-positioned to integrate these technologies.

Growing OBF Adoption: Each new app supporting OBF makes portability more practical and vendor lock-in less defensible.

Education Advocacy: With Education earning an F in the GPA, there is enormous room for policy advocacy around device access in schools and literacy instruction for AAC users.

Challenges

Community Growth: The project needs to expand its developer community beyond core maintainers. Better documentation, a documented API, and modernized tooling could help.

Stack Modernization: The Ember.js + Bower frontend may need updating to attract contributors and ensure long-term sustainability.

Identity Clarity: The Sweet Suite / CoughDrop / OpenAAC / Forbes AAC relationship needs clearer communication to avoid confusion among potential users and contributors.

Sweet Suite AAC is not the easiest AAC app to set up. It is not the one with the most GitHub stars. But it may be the most important. In a field where families can spend thousands of dollars on devices and software that lock them into a single vendor's ecosystem, Sweet Suite and OpenAAC represent a fundamentally different vision: one where communication tools are open, interoperable, community-governed, and available to everyone. The code is open. The standards are free. The question is whether the community will rally around it.
