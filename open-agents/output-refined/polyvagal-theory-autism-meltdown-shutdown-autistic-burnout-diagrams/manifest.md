## Diagram Descriptions

### Diagram 1: Three States, Three Epistemic Statuses
- **Type**: Comparison grid (custom SVG-style HTML)
- **Elements**: 4 columns × 6 rows = 18 content cells + 3 verdict badges
- **Key Insight**: The three states have three different evidence statuses. Meltdown maps onto measured autonomic data (Cheng 2020). Shutdown has no direct measurement and a contested mechanism. Burnout has the strongest empirical literature — but in an allostatic-load framework, not polyvagal.
- **Accessibility**: Grid uses both colour (green / amber / red tier) and explicit text labels for each tier; no information conveyed by colour alone.

### Diagram 2: Meltdown Phenomenology + Empirical Support
- **Type**: 5-step phenomenology cascade + 4-card evidence panel
- **Key Insight**: Cheng 2020 g = −0.72 RSA reactivity is the largest effect in autism HRV meta-analysis; consistent with sympathetic-over-arousal reframing but not a direct test of meltdown.
- **Accessibility**: Clear visual flow; verdict-strip summary at bottom.

### Diagram 3: Shutdown Phenomenology + "No In Vivo Measurement"
- **Type**: 5-step phenomenology cascade with Phung 2021 quotes + 4-claim examination panel
- **Key Insight**: Phenomenology is well-described; popular polyvagal mechanism (DMV-mediated freeze) is contested by Neuhuber & Berthoud 2022 comparative neuroanatomy; "dorsal vagal state" is not measured in vivo.
- **Accessibility**: Two-column layout with explicit "What still holds" green box for the phenomenology-is-real conclusion.

### Diagram 4: Burnout — Allostatic Load vs Polyvagal Reframing
- **Type**: Two-column framework comparison with 5 matched fields (core mechanism / operational definition / empirical literature / what the data fits / treatment implications)
- **Key Insight**: Allostatic load is the framework Raymaker's research actually uses. The polyvagal reframing of burnout as "chronic dorsal vagal state" is post-hoc and lacks direct measurement.
- **Accessibility**: Strong colour-coding (green for empirical, red for popular) with explicit verdict-strip below.

### Diagram 5: Burnout × PTSS × Repeated Meltdowns Overlap
- **Type**: Three-circle Venn (SVG) + per-construct annotation cards
- **Key Insight**: Substantial overlap (r = 0.70 burnout × PTSS) is not identity (49% shared variance). Distinct constructs, frequently co-occurring; treat each strand separately for cleaner intervention.
- **Accessibility**: Each correlation labelled explicitly in text; SVG has `<title>` and `<desc>` elements.

### Diagram 6: McEwen's Allostatic Load Curve
- **Type**: Curve diagram (SVG) over four stages + four stage-detail cards below
- **Key Insight**: The empirical framework for autistic burnout. Demand reduction (not acute regulation) is the lever that bends this curve.
- **Accessibility**: Curve gradient + four labelled background bands; stage cards repeat content in text.

### Diagram 7: Polyvagal Ladder Reframed for Autistic Experience
- **Type**: Three-rung ladder (HTML) + "IS / IS NOT" side panel
- **Key Insight**: The ladder can be useful scaffolding for self-narrative even if its mechanism claims are contested. The chapter respects users for whom this framing is helpful without endorsing the mechanism.
- **Accessibility**: Each rung clearly labelled top/middle/bottom; explicit IS/IS NOT distinction; no claim made implicitly.

### Diagram 8: Self-Narrative vs Diagnostic Substitute — Decision Aid
- **Type**: Decision tree from root question into two coloured branches with 5 use-cases each
- **Key Insight**: Scaffolding adds a way of speaking about your state. Substitute replaces the question of what's actually going on. The line is crossed when "I'm in dorsal vagal" stops further inquiry.
- **Accessibility**: Each use-case has explicit next-step pointer for follow-up action.

### Diagram 9: Cross-Reference Map
- **Type**: Network diagram (SVG) with 4 connected nodes + 4 legend cards
- **Key Insight**: This chapter sits at an intersection. Following the cross-references makes the deep dive denser without bloating any one chapter.
- **Accessibility**: Each edge labelled in text; legend cards repeat URL paths.
